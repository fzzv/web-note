# Wails v3 新特性

> Wails v3 目前处于 Alpha 阶段，API 已趋于稳定，已有项目在生产环境运行。
> 本章基于 v3 Alpha 版本编写，部分 API 在正式发布时可能有微调。

---

## 1. v3 核心变化概览

### 1.1 v2 vs v3 对比总览

| 特性 | v2 | v3 |
|------|----|----|
| API 风格 | 声明式（一次性配置） | 过程式（逐步构建） |
| 窗口 | 单窗口 | **多窗口** |
| 系统托盘 | 不支持 | **原生支持** |
| 绑定方式 | `Bind: []interface{}` | `Services` |
| 生命周期 | 回调函数在 options 中 | Service 接口方法 |
| 窗口操作 | `runtime.Window*(ctx)` | `window.Method()` |
| 事件系统 | `runtime.Events*(ctx)` | `app.Event.*` |
| 对话框 | `runtime.OpenFileDialog(ctx)` | `app.Dialog.*` |
| 菜单 | `menu.NewMenu()` | `app.Menu.New()` |
| 构建系统 | Wails CLI 内置 | **Taskfile**（可自定义） |
| 绑定生成 | 运行时反射 | **静态分析**（编译期） |

### 1.2 最小应用对比

**v2 — 声明式，所有配置在一个结构体中**：

```go
package main

import (
    "context"
    "embed"
    "github.com/wailsapp/wails/v2"
    "github.com/wailsapp/wails/v2/pkg/options"
    "github.com/wailsapp/wails/v2/pkg/options/assetserver"
)

//go:embed all:frontend/dist
var assets embed.FS

type App struct {
    ctx context.Context
}

func (a *App) startup(ctx context.Context) { a.ctx = ctx }

func (a *App) Greet(name string) string {
    return "Hello " + name
}

func main() {
    app := &App{}
    wails.Run(&options.App{
        Title:  "My App",
        Width:  1024,
        Height: 768,
        AssetServer: &assetserver.Options{Assets: assets},
        OnStartup: app.startup,
        Bind:       []interface{}{app},
    })
}
```

**v3 — 过程式，分步创建应用、窗口、服务**：

```go
package main

import (
    _ "embed"
    "fmt"
    "log"
    "github.com/wailsapp/wails/v3/pkg/application"
)

//go:embed frontend/dist
var assets embed.FS

type GreetService struct{}

func (g *GreetService) Greet(name string) string {
    result := "Hello " + name
    fmt.Printf("Greet 调用: %s → %s\n", name, result)
    // 输出: Greet 调用: World → Hello World
    return result
}

func main() {
    // 1. 创建应用
    app := application.New(application.Options{
        Name: "My App",
        Services: []application.Service{
            application.NewService(&GreetService{}),
        },
        Assets: application.AssetOptions{
            Handler: application.AssetFileServerFS(assets),
        },
    })

    // 2. 创建窗口
    window := app.Window.NewWithOptions(application.WebviewWindowOptions{
        Title:  "My App",
        Width:  1024,
        Height: 768,
    })
    window.Centre()

    fmt.Println("应用启动中...")
    // 输出: 应用启动中...

    // 3. 运行应用
    err := app.Run()
    if err != nil {
        log.Fatal(err)
    }
}
```

**核心区别**：
- 应用创建（`application.New`）和窗口创建（`app.Window.NewWithOptions`）分离
- `Bind` 被 `Services` 取代
- 窗口是独立对象，可以创建多个
- `app.Run()` 显式启动事件循环

## 2. Services（服务）

Services 是 v3 中取代 v2 `Bind` 的核心概念。每个 Service 是一个普通的 Go 结构体，通过实现特定接口获得生命周期管理能力。

### 2.1 基本 Service

```go
// 最简单的 Service — 只需要是一个结构体，公开方法自动暴露给前端
type CalculatorService struct{}

func (c *CalculatorService) Add(a, b float64) float64 {
    result := a + b
    fmt.Printf("计算: %.2f + %.2f = %.2f\n", a, b, result)
    // 输出: 计算: 3.00 + 5.00 = 8.00
    return result
}

func (c *CalculatorService) Multiply(a, b float64) float64 {
    result := a * b
    fmt.Printf("计算: %.2f × %.2f = %.2f\n", a, b, result)
    // 输出: 计算: 3.00 × 5.00 = 15.00
    return result
}
```

注册：

```go
app := application.New(application.Options{
    Name: "Calculator",
    Services: []application.Service{
        application.NewService(&CalculatorService{}),
    },
})
```

### 2.2 Service 生命周期

v3 的 Service 通过实现接口来参与生命周期，不再需要手动保存 `context`：

```go
import (
    "context"
    "database/sql"
    "fmt"
    "github.com/wailsapp/wails/v3/pkg/application"
)

type DBService struct {
    db *sql.DB
}

// ServiceStartup — 应用启动时调用（取代 v2 的 OnStartup）
// 返回 error 会阻止应用启动
func (s *DBService) ServiceStartup(ctx context.Context, options application.ServiceOptions) error {
    fmt.Println("[DBService] 启动中...")
    // 输出: [DBService] 启动中...

    var err error
    s.db, err = sql.Open("sqlite", "app.db")
    if err != nil {
        fmt.Printf("[DBService] 数据库连接失败: %v\n", err)
        // 输出: [DBService] 数据库连接失败: ...
        return err // 返回错误会阻止应用启动
    }

    fmt.Println("[DBService] 数据库连接成功")
    // 输出: [DBService] 数据库连接成功
    return nil
}

// ServiceShutdown — 应用关闭时调用（取代 v2 的 OnShutdown）
func (s *DBService) ServiceShutdown() error {
    if s.db != nil {
        s.db.Close()
        fmt.Println("[DBService] 数据库连接已关闭")
        // 输出: [DBService] 数据库连接已关闭
    }
    return nil
}

// ServiceName — 可选，自定义服务名称（默认使用结构体名）
func (s *DBService) ServiceName() string {
    return "database"
}

// 业务方法 — 自动暴露给前端
func (s *DBService) GetUser(id int) (string, error) {
    var name string
    err := s.db.QueryRow("SELECT name FROM users WHERE id = ?", id).Scan(&name)
    if err != nil {
        return "", fmt.Errorf("用户不存在: %w", err)
    }
    fmt.Printf("[DBService] 查询用户: id=%d, name=%s\n", id, name)
    // 输出: [DBService] 查询用户: id=1, name=张三
    return name, nil
}
```

**Service 生命周期接口一览**：

| 接口 | 方法签名 | 触发时机 |
|------|----------|----------|
| `ServiceStartup` | `ServiceStartup(ctx, options) error` | 应用启动时 |
| `ServiceShutdown` | `ServiceShutdown() error` | 应用关闭时 |
| `ServiceName` | `ServiceName() string` | 获取服务名称 |

### 2.3 注册多个 Service

```go
app := application.New(application.Options{
    Name: "My App",
    Services: []application.Service{
        application.NewService(&GreetService{}),
        application.NewService(&DBService{}),
        application.NewService(&FileService{}),
        application.NewService(&ConfigService{}),
    },
})
```

v3 中 Service 是完全独立的 Go 结构体，**无需保存 context**，更容易进行单元测试：

```go
// 测试 Service（完全不依赖 Wails 运行时）
func TestGreetService(t *testing.T) {
    svc := &GreetService{}
    result := svc.Greet("Test")
    if result != "Hello Test" {
        t.Errorf("期望 'Hello Test', 得到 '%s'", result)
    }
    // 输出: --- PASS: TestGreetService (0.00s)
}
```

## 3. 多窗口支持

v3 最重要的新特性 — 原生支持多个独立窗口。

### 3.1 创建多个窗口

```go
package main

import (
    "fmt"
    "log"
    "github.com/wailsapp/wails/v3/pkg/application"
)

type App struct {
    app            *application.Application
    mainWindow     *application.WebviewWindow
    settingsWindow *application.WebviewWindow
    previewWindow  *application.WebviewWindow
}

func main() {
    myApp := &App{}

    myApp.app = application.New(application.Options{
        Name: "多窗口演示",
        Services: []application.Service{
            application.NewService(myApp),
        },
    })

    // 主窗口
    myApp.mainWindow = myApp.app.Window.NewWithOptions(application.WebviewWindowOptions{
        Name:   "main",
        Title:  "主窗口 - 编辑器",
        Width:  1200,
        Height: 800,
    })
    myApp.mainWindow.Centre()
    fmt.Println("主窗口已创建")
    // 输出: 主窗口已创建

    // 设置窗口（初始隐藏）
    myApp.settingsWindow = myApp.app.Window.NewWithOptions(application.WebviewWindowOptions{
        Name:      "settings",
        Title:     "设置",
        Width:     600,
        Height:    400,
        Hidden:    true,
        Resizable: false,
    })
    fmt.Println("设置窗口已创建（隐藏）")
    // 输出: 设置窗口已创建（隐藏）

    // 预览窗口（初始隐藏）
    myApp.previewWindow = myApp.app.Window.NewWithOptions(application.WebviewWindowOptions{
        Name:   "preview",
        Title:  "预览",
        Width:  800,
        Height: 600,
        Hidden: true,
    })
    fmt.Println("预览窗口已创建（隐藏）")
    // 输出: 预览窗口已创建（隐藏）

    err := myApp.app.Run()
    if err != nil {
        log.Fatal(err)
    }
}

// 显示设置窗口（前端调用）
func (a *App) ShowSettings() {
    a.settingsWindow.Show()
    a.settingsWindow.SetFocus()
    fmt.Println("设置窗口已显示")
    // 输出: 设置窗口已显示
}

// 隐藏设置窗口
func (a *App) HideSettings() {
    a.settingsWindow.Hide()
    fmt.Println("设置窗口已隐藏")
    // 输出: 设置窗口已隐藏
}

// 显示/隐藏预览窗口
func (a *App) TogglePreview() {
    if a.previewWindow.IsVisible() {
        a.previewWindow.Hide()
        fmt.Println("预览窗口已隐藏")
        // 输出: 预览窗口已隐藏
    } else {
        a.previewWindow.Show()
        a.previewWindow.SetFocus()
        fmt.Println("预览窗口已显示")
        // 输出: 预览窗口已显示
    }
}
```

### 3.2 窗口 API（对象方法，非全局函数）

v3 中窗口操作直接调用窗口对象的方法，不再需要传递 `context`：

```go
// ===== v2 — 全局函数 + context =====
runtime.WindowSetTitle(ctx, "新标题")
runtime.WindowSetSize(ctx, 800, 600)
runtime.WindowCenter(ctx)
runtime.WindowMaximise(ctx)

// ===== v3 — 窗口对象方法 =====
window.SetTitle("新标题")
window.SetSize(800, 600)
window.Centre()
window.Maximise()
```

**v3 窗口方法一览**：

```go
// 大小与位置
window.SetSize(width, height int)
window.SetMinSize(width, height int)
window.SetMaxSize(width, height int)
window.SetPosition(x, y int)
window.Centre()

// 状态控制
window.Show()
window.Hide()
window.Maximise()
window.UnMaximise()
window.Minimise()
window.UnMinimise()
window.Fullscreen()
window.UnFullscreen()
window.SetFocus()

// 属性
window.SetTitle(title string)
window.SetAlwaysOnTop(onTop bool)
window.SetResizable(resizable bool)

// 状态查询
window.IsVisible() bool
window.IsMaximised() bool
window.IsMinimised() bool
window.IsFullscreen() bool

// 内容
window.SetURL(url string)           // 加载指定 URL
window.ExecJS(js string)            // 执行 JS 代码
window.Reload()                      // 刷新页面

// 事件（窗口级别的事件发射）
window.EmitEvent(name string, data ...interface{})
```

### 3.3 窗口事件处理

v3 提供了丰富的窗口级别事件回调：

```go
func setupWindowEvents(window *application.WebviewWindow) {
    // 焦点事件
    window.OnFocus(func() {
        fmt.Println("窗口获得焦点")
        // 输出: 窗口获得焦点
    })

    window.OnBlur(func() {
        fmt.Println("窗口失去焦点")
        // 输出: 窗口失去焦点
    })

    // 状态变化事件
    window.OnMaximise(func() {
        fmt.Println("窗口最大化")
        // 输出: 窗口最大化
    })

    window.OnUnMaximise(func() {
        fmt.Println("窗口恢复")
        // 输出: 窗口恢复
    })

    window.OnMinimise(func() {
        fmt.Println("窗口最小化")
        // 输出: 窗口最小化
    })

    window.OnFullscreen(func() {
        fmt.Println("窗口全屏")
        // 输出: 窗口全屏
    })

    // 大小和位置变化
    window.OnResize(func(width, height int) {
        fmt.Printf("窗口大小变化: %dx%d\n", width, height)
        // 输出: 窗口大小变化: 1024x768
    })

    window.OnMove(func(x, y int) {
        fmt.Printf("窗口位置变化: (%d, %d)\n", x, y)
        // 输出: 窗口位置变化: (200, 150)
    })

    // 关闭事件（返回 true 允许关闭，false 阻止关闭）
    window.OnClose(func() bool {
        fmt.Println("窗口即将关闭")
        // 输出: 窗口即将关闭
        return true // 允许关闭
    })

    // 销毁事件（窗口已关闭，无法阻止）
    window.OnDestroy(func() {
        fmt.Println("窗口已销毁")
        // 输出: 窗口已销毁
    })
}
```

### 3.4 案例：窗口状态持久化

```go
type WindowState struct {
    X          int  `json:"x"`
    Y          int  `json:"y"`
    Width      int  `json:"width"`
    Height     int  `json:"height"`
    Maximised  bool `json:"maximised"`
}

type ManagedWindow struct {
    window *application.WebviewWindow
    state  WindowState
    dirty  bool
}

func NewManagedWindow(app *application.Application) *ManagedWindow {
    mw := &ManagedWindow{}

    mw.window = app.Window.NewWithOptions(application.WebviewWindowOptions{
        Name:   "main",
        Title:  "My App",
        Width:  800,
        Height: 600,
    })

    // 加载上次的窗口位置
    mw.loadState()

    // 监听窗口变化
    mw.window.OnMove(func(x, y int) {
        mw.state.X = x
        mw.state.Y = y
        mw.dirty = true
    })

    mw.window.OnResize(func(w, h int) {
        mw.state.Width = w
        mw.state.Height = h
        mw.dirty = true
    })

    mw.window.OnMaximise(func() {
        mw.state.Maximised = true
        mw.dirty = true
    })

    mw.window.OnUnMaximise(func() {
        mw.state.Maximised = false
        mw.dirty = true
    })

    // 关闭时保存
    mw.window.OnDestroy(func() {
        if mw.dirty {
            mw.saveState()
            fmt.Println("窗口状态已保存")
            // 输出: 窗口状态已保存
        }
    })

    return mw
}

func (mw *ManagedWindow) loadState() {
    data, err := os.ReadFile("window-state.json")
    if err != nil {
        fmt.Println("使用默认窗口位置")
        // 输出: 使用默认窗口位置
        return
    }

    json.Unmarshal(data, &mw.state)

    mw.window.SetPosition(mw.state.X, mw.state.Y)
    mw.window.SetSize(mw.state.Width, mw.state.Height)
    if mw.state.Maximised {
        mw.window.Maximise()
    }

    fmt.Printf("恢复窗口状态: %dx%d at (%d,%d)\n",
        mw.state.Width, mw.state.Height, mw.state.X, mw.state.Y)
    // 输出: 恢复窗口状态: 1200x800 at (100,50)
}

func (mw *ManagedWindow) saveState() {
    data, _ := json.Marshal(mw.state)
    os.WriteFile("window-state.json", data, 0644)
}
```

### 3.5 窗口间通信

通过应用级事件系统在窗口之间传递数据：

```go
// ===== 主窗口中的操作 =====
func (a *App) UpdateContent(content string) {
    fmt.Printf("内容更新: %s\n", content[:min(50, len(content))])
    // 输出: 内容更新: # Hello World...

    // 通过应用事件广播给所有窗口
    a.app.Event.Emit("content:updated", map[string]interface{}{
        "content": content,
        "source":  "main",
    })
}

// ===== 预览窗口监听 =====
func (a *App) setupPreviewWindow() {
    a.app.Event.On("content:updated", func(event *application.WailsEvent) {
        data := event.Data.(map[string]interface{})
        content := data["content"].(string)
        source := data["source"].(string)

        fmt.Printf("[预览窗口] 收到来自 %s 的内容更新 (%d 字)\n", source, len(content))
        // 输出: [预览窗口] 收到来自 main 的内容更新 (256 字)

        // 向预览窗口的前端推送
        a.previewWindow.EmitEvent("preview:refresh", content)
    })
}
```

前端（预览窗口）：

```typescript
// 监听窗口级别的事件
wails.Events.On("preview:refresh", (content: string) => {
    console.log("预览内容更新:", content.substring(0, 50))
    // 输出: 预览内容更新: # Hello World...
    renderMarkdown(content)
})
```

## 4. 系统托盘（System Tray）

v3 新增的系统托盘功能，v2 中不支持。

### 4.1 基本系统托盘

```go
package main

import (
    _ "embed"
    "fmt"
    "log"
    "runtime"

    "github.com/wailsapp/wails/v3/pkg/application"
    "github.com/wailsapp/wails/v3/pkg/icons"
)

//go:embed icon.png
var appIcon []byte

func main() {
    app := application.New(application.Options{
        Name: "托盘应用",
        // macOS: 不在 Dock 中显示图标
        Mac: application.MacOptions{
            ActivationPolicy: application.ActivationPolicyAccessory,
        },
    })

    // 创建主窗口（初始隐藏）
    mainWindow := app.Window.NewWithOptions(application.WebviewWindowOptions{
        Name:   "main",
        Title:  "我的托盘应用",
        Width:  800,
        Height: 600,
        Hidden: true, // 启动时隐藏
        Windows: application.WindowsWindow{
            HiddenOnTaskbar: true, // Windows 任务栏中不显示
        },
    })

    // ===== 创建系统托盘 =====
    systray := app.SystemTray.New()

    // 设置托盘图标（区分平台）
    if runtime.GOOS == "darwin" {
        systray.SetTemplateIcon(icons.SystrayMacTemplate)
        fmt.Println("设置 macOS 模板图标")
        // 输出: 设置 macOS 模板图标
    } else {
        systray.SetIcon(icons.SystrayLight)          // 浅色模式图标
        systray.SetDarkModeIcon(icons.SystrayDark)   // 深色模式图标
        fmt.Println("设置 Windows/Linux 图标")
        // 输出: 设置 Windows/Linux 图标
    }

    // 设置托盘标签（macOS 显示在图标旁边）
    systray.SetLabel("My App")

    // ===== 创建托盘菜单 =====
    trayMenu := app.Menu.New()

    trayMenu.Add("显示窗口").OnClick(func(ctx *application.Context) {
        mainWindow.Show()
        mainWindow.SetFocus()
        mainWindow.Centre()
        fmt.Println("显示主窗口")
        // 输出: 显示主窗口
    })

    trayMenu.Add("隐藏窗口").OnClick(func(ctx *application.Context) {
        mainWindow.Hide()
        fmt.Println("隐藏主窗口")
        // 输出: 隐藏主窗口
    })

    trayMenu.AddSeparator()

    trayMenu.Add("关于").OnClick(func(ctx *application.Context) {
        fmt.Println("关于被点击")
        // 输出: 关于被点击
    })

    trayMenu.AddSeparator()

    trayMenu.Add("退出").OnClick(func(ctx *application.Context) {
        fmt.Println("应用退出")
        // 输出: 应用退出
        app.Quit()
    })

    systray.SetMenu(trayMenu)

    fmt.Println("系统托盘已创建")
    // 输出: 系统托盘已创建

    err := app.Run()
    if err != nil {
        log.Fatal(err)
    }
}
```

### 4.2 托盘附加窗口（点击图标弹出窗口）

一种常见模式：点击托盘图标弹出一个小窗口（类似微信、Slack）：

```go
// 创建弹出窗口（无边框、置顶、初始隐藏）
popupWindow := app.Window.NewWithOptions(application.WebviewWindowOptions{
    Name:        "popup",
    Width:       400,
    Height:      500,
    Frameless:   true,
    AlwaysOnTop: true,
    Hidden:      true,
    Windows: application.WindowsWindow{
        HiddenOnTaskbar: true,
    },
})

// 将窗口附加到托盘图标
// - 点击托盘图标时自动显示窗口（居中于图标下方）
// - 窗口失去焦点时自动隐藏
systray.AttachWindow(popupWindow).WindowOffset(5)

fmt.Println("弹出窗口已附加到托盘图标")
// 输出: 弹出窗口已附加到托盘图标
```

`AttachWindow` 的行为：
- 点击托盘图标 → 窗口自动出现在图标附近
- `WindowOffset(5)` → 窗口与图标之间有 5px 间距
- 窗口失去焦点 → 自动隐藏
- 再次点击托盘图标 → 重新显示

## 5. 事件系统

### 5.1 应用级事件

v3 的事件 API 挂在 `app.Event` 上，不再需要传递 `context`：

```go
// ===== v2 =====
runtime.EventsEmit(ctx, "eventName", data)
runtime.EventsOn(ctx, "eventName", callback)

// ===== v3 =====
app.Event.Emit("eventName", data)
app.Event.On("eventName", callback)
```

**v3 事件 API**：

```go
// 发送事件
app.Event.Emit("user:login", map[string]interface{}{
    "name": "张三",
    "role": "admin",
})
fmt.Println("已发送登录事件")
// 输出: 已发送登录事件

// 持续监听
app.Event.On("user:login", func(event *application.WailsEvent) {
    data := event.Data.(map[string]interface{})
    fmt.Printf("用户登录: %s (%s)\n", data["name"], data["role"])
    // 输出: 用户登录: 张三 (admin)
})

// 只监听一次
app.Event.Once("app:initialized", func(event *application.WailsEvent) {
    fmt.Println("应用初始化完成（只触发一次）")
    // 输出: 应用初始化完成（只触发一次）
})

// 取消监听（On 返回取消函数）
cancel := app.Event.On("tick", func(event *application.WailsEvent) {
    fmt.Println("tick")
})
// 稍后取消
cancel()
fmt.Println("tick 事件已取消监听")
// 输出: tick 事件已取消监听
```

### 5.2 窗口级事件

每个窗口可以独立发射事件：

```go
// 从特定窗口发射事件（只有该窗口的前端能收到）
mainWindow.EmitEvent("window:notification", map[string]interface{}{
    "title":   "提示",
    "message": "文件已保存",
})
fmt.Println("向主窗口发送了通知")
// 输出: 向主窗口发送了通知

// 也可以从窗口发射应用级事件（所有窗口都能收到）
app.Event.Emit("global:theme-changed", "dark")
fmt.Println("全局主题变更事件已广播")
// 输出: 全局主题变更事件已广播
```

### 5.3 系统事件

v3 新增了系统级事件的监听能力：

```go
// 监听系统主题变化
app.Event.On("system:theme-changed", func(event *application.WailsEvent) {
    isDark := event.Data.(bool)
    fmt.Printf("系统主题变化: 深色模式=%v\n", isDark)
    // 输出: 系统主题变化: 深色模式=true
})

// 应用级生命周期事件
app.Event.On("application:started", func(event *application.WailsEvent) {
    fmt.Println("应用已启动")
    // 输出: 应用已启动
})
```

## 8. 构建系统

### 8.1 Taskfile 取代内置构建

v3 使用 [Taskfile](https://taskfile.dev/) 作为构建系统，取代 v2 内置的不透明构建流程。

**项目根目录的 `Taskfile.yml`**：

```yaml
version: '3'

includes:
  common: ./build/Taskfile.yml
  windows: ./build/windows/Taskfile.yml
  darwin:  ./build/darwin/Taskfile.yml
  linux:   ./build/linux/Taskfile.yml

vars:
  APP_NAME: "myapp"
  BIN_DIR: "bin"
  VITE_PORT: '{{.WAILS_VITE_PORT | default 9245}}'

tasks:
  build:
    summary: 构建应用
    cmds:
      - task: "{{OS}}:build"

  package:
    summary: 打包生产版本
    cmds:
      - task: "{{OS}}:package"

  run:
    summary: 运行应用
    cmds:
      - task: "{{OS}}:run"

  dev:
    summary: 开发模式
    cmds:
      - wails3 dev -config ./build/config.yml -port {{.VITE_PORT}}
```

**常用命令**：

```bash
# 安装 Task 工具
go install github.com/go-task/task/v3/cmd/task@latest

# 开发模式
task dev

# 构建
task build

# 打包
task package

# 运行
task run
```

```
$ task build
task: [windows:build] go build -o bin/myapp.exe .
task: 构建完成

$ task dev
task: [dev] wails3 dev -config ./build/config.yml -port 9245
  → Vite dev server: http://localhost:9245
  → Application running...
```

### 8.2 自定义构建流程

Taskfile 的最大优势是**完全透明和可自定义**：

```yaml
# build/windows/Taskfile.yml
version: '3'

tasks:
  build:
    cmds:
      # 你可以在这里插入任何自定义步骤
      - echo "生成版本信息..."
      - go generate ./...
      - echo "编译 Windows 版本..."
      - go build -trimpath -ldflags "-s -w -H windowsgui" -o {{.BIN_DIR}}/{{.APP_NAME}}.exe .
      - echo "构建完成!"

  package:
    deps: [build]
    cmds:
      - echo "生成 NSIS 安装包..."
      - makensis build/windows/installer/project.nsi
```

### 8.3 绑定生成（静态分析）

v3 使用静态分析而非运行时反射来生成绑定代码，速度更快且更可靠：

```bash
# 生成绑定（分析 Go 源码中的 Service 方法）
wails3 generate bindings
```

```
Analysing project...
  Found service: GreetService (2 methods)
  Found service: DBService (3 methods)
  Found service: FileService (4 methods)
Generating bindings...
  → frontend/bindings/GreetService.ts
  → frontend/bindings/DBService.ts
  → frontend/bindings/FileService.ts
Done in 0.8s
```

相比 v2 的运行时反射：
- 编译期发现类型错误（更安全）
- 生成速度更快
- 不依赖运行应用

## 10. v2 → v3 迁移指南

### 10.1 迁移步骤

```
1. 更新 import 路径
   github.com/wailsapp/wails/v2  →  github.com/wailsapp/wails/v3

2. wails.Run(&options.App{})  →  application.New() + app.Run()

3. Bind  →  Services

4. OnStartup/OnShutdown  →  ServiceStartup/ServiceShutdown

5. runtime.Window*(ctx)  →  window.Method()

6. runtime.Events*(ctx)  →  app.Event.*

7. runtime.OpenFileDialog(ctx)  →  app.Dialog.OpenFile()

8. menu.NewMenu()  →  app.Menu.New()

9. wails build  →  task build

10. 新增: 多窗口、系统托盘、上下文菜单
```

### 10.2 逐项对照

**应用初始化**：

```go
// v2
wails.Run(&options.App{
    Title:     "My App",
    Width:     1024,
    Height:    768,
    OnStartup: app.startup,
    Bind:      []interface{}{app},
})

// v3
app := application.New(application.Options{
    Name: "My App",
    Services: []application.Service{
        application.NewService(&MyService{}),
    },
})
window := app.Window.NewWithOptions(application.WebviewWindowOptions{
    Title: "My App", Width: 1024, Height: 768,
})
app.Run()
```

**生命周期**：

```go
// v2 — 在 options 中指定回调函数
func (a *App) startup(ctx context.Context) { a.ctx = ctx }
func (a *App) shutdown(ctx context.Context) { /* cleanup */ }

// v3 — 实现 Service 接口
func (s *MyService) ServiceStartup(ctx context.Context, opts application.ServiceOptions) error {
    return nil
}
func (s *MyService) ServiceShutdown() error {
    return nil
}
```

**窗口操作**：

```go
// v2
runtime.WindowSetTitle(ctx, "新标题")
runtime.WindowSetSize(ctx, 800, 600)
runtime.WindowCenter(ctx)

// v3
window.SetTitle("新标题")
window.SetSize(800, 600)
window.Centre()
```

**事件**：

```go
// v2
runtime.EventsEmit(ctx, "myEvent", data)
runtime.EventsOn(ctx, "myEvent", func(args ...interface{}) { })
runtime.EventsOff(ctx, "myEvent")

// v3
app.Event.Emit("myEvent", data)
cancel := app.Event.On("myEvent", func(e *application.WailsEvent) { })
cancel() // 取消监听
```

**对话框**：

```go
// v2
runtime.OpenFileDialog(ctx, runtime.OpenDialogOptions{Title: "打开"})
runtime.MessageDialog(ctx, runtime.MessageDialogOptions{Title: "提示"})

// v3
app.Dialog.OpenFile(application.OpenFileDialogOptions{Title: "打开"})
app.Dialog.Message(application.MessageDialogOptions{Title: "提示"})
```

## 小结

| v3 新特性 | 核心要点 |
|-----------|----------|
| 过程式 API | `application.New()` + `app.Window.New()` + `app.Run()` 分步构建 |
| Services | 取代 `Bind`，通过实现接口获得生命周期，更易测试 |
| 多窗口 | `app.Window.NewWithOptions()` 创建任意多个独立窗口 |
| 窗口事件 | `OnFocus/OnBlur/OnResize/OnMove/OnClose/OnDestroy` 完整回调 |
| 窗口通信 | `app.Event.Emit()` 广播 + `window.EmitEvent()` 窗口级别 |
| 系统托盘 | `app.SystemTray.New()`，支持菜单、图标、附加窗口 |
| 事件系统 | `app.Event.On/Once/Emit`，`On` 返回取消函数 |
| 对话框 | `app.Dialog.OpenFile/SaveFile/Message`，无需 context |
| 菜单 | `app.Menu.New()`，链式调用 `.Add().SetAccelerator().OnClick()` |
| 上下文菜单 | `app.ContextMenu.New()`，HTML `data-contextmenu` 属性触发 |
| 构建系统 | Taskfile 取代内置构建，完全透明可自定义 |
| 绑定生成 | 静态分析取代运行时反射，编译期类型安全 |
