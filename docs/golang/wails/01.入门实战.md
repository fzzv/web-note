# Wails 入门实战

---

## 1. Wails CLI 与第一个应用

### 1.1 CLI 常用命令

```bash
# 检查开发环境是否就绪
wails doctor

# 创建新项目（-n 项目名，-t 模板）
wails init -n myapp -t vue-ts

# 开发模式（热重载）
wails dev

# 构建生产版本
wails build

# 生成 Go 绑定的 JS/TS 模块
wails generate module
```

**可用模板**（通过 `wails init -l` 查看完整列表）：

| 模板名 | 说明 |
|--------|------|
| `vanilla` | 原生 HTML/JS/CSS |
| `vue` | Vue 3 |
| `vue-ts` | Vue 3 + TypeScript（推荐） |
| `react` | React |
| `react-ts` | React + TypeScript |
| `svelte` | Svelte |
| `svelte-ts` | Svelte + TypeScript |
| `preact` | Preact |
| `lit` | Lit |

`wails init` 还支持以下选项：

```bash
wails init -n myapp -t vue-ts   # 指定模板
            -d ./projects       # 指定输出目录
            -g                  # 初始化 git 仓库
            -ide vscode         # 生成 VS Code 项目配置
            -ide goland         # 生成 GoLand 项目配置
```

### 1.2 创建并运行第一个项目

```bash
# 1. 创建项目
wails init -n hello-wails -t vue-ts

# 2. 进入项目目录
cd hello-wails

# 3. 启动开发模式
wails dev
```

`wails dev` 启动后会：
- 编译 Go 后端代码
- 启动 Vite 开发服务器（前端热重载）
- 打开一个原生窗口展示应用
- 修改 Go 代码后自动重新编译
- 修改前端代码后即时刷新

> `wails dev` 还支持 `-browser` 参数，在浏览器中打开应用方便调试。


**关键文件说明**：

| `frontend/wailsjs/` | **自动生成**，不要手动修改。包含 Go 方法的 JS 包装和运行时 API |

## 2. 核心概念：Go 与前端的桥接（Bindings）

Wails 最核心的能力就是让前端 JavaScript 可以**直接调用 Go 方法**，就像调用本地函数一样。这个机制叫做**绑定（Binding）**。

### 2.1 工作原理

```
┌─────────────────────────────────────────────────┐
│                   Wails 应用                     │
│                                                  │
│  ┌──────────────┐          ┌──────────────────┐  │
│  │   Go 后端     │  绑定    │   前端（WebView） │  │
│  │              │ ◄──────► │                  │  │
│  │  App.Greet() │          │  Greet("Tom")    │  │
│  │  App.GetData()│         │  GetData()       │  │
│  └──────────────┘          └──────────────────┘  │
│         │                           │            │
│         └── IPC 通信（JSON 序列化）──┘            │
└─────────────────────────────────────────────────┘
```

**核心流程**：

1. Go 端定义结构体方法（**公开方法**，即首字母大写）
2. 在 `main.go` 中通过 `Bind` 注册结构体实例
3. Wails 自动在 `frontend/wailsjs/go/` 下生成对应的 JS/TS 包装函数
4. 前端通过 `import` 导入这些函数，直接调用（返回 `Promise`）
5. 底层通过 IPC（进程间通信）+ JSON 序列化完成数据传递

### 2.2 Go 后端：定义可绑定的结构体

`app.go` — 应用核心逻辑：

```go
package main

import (
    "context"
    "fmt"
)

// App 结构体 — 承载应用的业务逻辑
type App struct {
    ctx context.Context // Wails 运行时上下文，用于调用运行时 API
}

// NewApp 创建 App 实例
func NewApp() *App {
    return &App{}
}

// startup 生命周期钩子 — 应用启动时调用
// 保存 context 以便后续调用运行时 API（对话框、事件等）
func (a *App) startup(ctx context.Context) {
    a.ctx = ctx
}

// Greet ⭐ 公开方法 — 会自动暴露给前端
// 规则：方法名首字母必须大写
func (a *App) Greet(name string) string {
    return fmt.Sprintf("Hello %s, It's show time!", name)
}
```

**绑定规则**：

| 规则 | 说明 |
|------|------|
| 方法名首字母**大写** | `Greet` ✅ 会被绑定；`greet` ❌ 不会 |
| 接收者必须是**指针** | `func (a *App)` ✅；`func (a App)` ❌ |
| 参数和返回值必须可 JSON 序列化 | 基本类型、struct、slice、map ✅；chan、func ❌ |
| 最多返回两个值 | `(string, error)` ✅；前端收到 error 时 Promise reject |

### 2.3 注册绑定并启动应用

`main.go` — 应用入口：

```go
package main

import (
    "embed"
    "log"

    "github.com/wailsapp/wails/v2"
    "github.com/wailsapp/wails/v2/pkg/options"
    "github.com/wailsapp/wails/v2/pkg/options/assetserver"
)

// 使用 Go embed 将前端构建产物嵌入二进制文件
//go:embed all:frontend/dist
var assets embed.FS

func main() {
    // 创建 App 实例
    app := NewApp()

    // 配置并启动 Wails 应用
    err := wails.Run(&options.App{
        Title:  "Hello Wails",       // 窗口标题
        Width:  1024,                // 窗口宽度
        Height: 768,                 // 窗口高度
        AssetServer: &assetserver.Options{
            Assets: assets,          // 嵌入的前端资源
        },
        BackgroundColour: &options.RGBA{R: 27, G: 38, B: 54, A: 1},
        OnStartup:  app.startup,     // 注册生命周期钩子
        Bind: []interface{}{
            app,                     // ⭐ 注册绑定 — 将 app 的所有公开方法暴露给前端
        },
    })

    if err != nil {
        log.Fatal(err)
    }
}
```

**关键解析**：

- `//go:embed all:frontend/dist` — 将前端构建产物（HTML/CSS/JS）嵌入到 Go 二进制中，最终只需分发一个可执行文件
- `Bind: []interface{}{app}` — **核心！** 将 `app` 实例注册为绑定对象，其所有公开方法都会被暴露
- 可以绑定**多个结构体**：`Bind: []interface{}{app, userService, fileService}`

### 2.4 前端调用 Go 方法

Wails 会在 `frontend/wailsjs/go/main/` 目录下自动生成对应的 JS/TS 文件。

**自动生成的文件 `frontend/wailsjs/go/main/App.js`**：

```javascript
// 自动生成 — 不要手动修改
export function Greet(arg1) {
    return window['go']['main']['App']['Greet'](arg1);
}
```

**自动生成的类型定义 `frontend/wailsjs/go/main/App.d.ts`**：

```typescript
export function Greet(arg1: string): Promise<string>;
```

**在 Vue 组件中调用**：

```vue
<script setup lang="ts">
import { ref } from 'vue'
import { Greet } from '../wailsjs/go/main/App'

const name = ref('')
const greeting = ref('')

async function greet() {
    // 调用 Go 的 App.Greet 方法 — 返回 Promise
    greeting.value = await Greet(name.value)
}
</script>

<template>
    <div>
        <input v-model="name" placeholder="输入你的名字" />
        <button @click="greet">打招呼</button>
        <p>{{ greeting }}</p>
    </div>
</template>
```

### 2.5 Go 与 TypeScript 数据类型映射

当绑定方法的参数或返回值包含 **struct** 时，Wails 会自动生成对应的 TypeScript class。

**Go 端定义**：

```go
type Address struct {
    Street   string `json:"street"`
    Postcode string `json:"postcode"`
}

type Person struct {
    Name    string   `json:"name"`
    Age     int      `json:"age"`
    Address *Address `json:"address"`
}

// 使用 struct 作为参数和返回值
func (a *App) CreatePerson(name string, age int) Person {
    return Person{
        Name: name,
        Age:  age,
        Address: &Address{
            Street:   "123 Main St",
            Postcode: "12345",
        },
    }
}

func (a *App) GetPersonInfo(p Person) string {
    return fmt.Sprintf("%s, %d years old, lives at %s", p.Name, p.Age, p.Address.Street)
}
```

**自动生成的 TypeScript class**（`frontend/wailsjs/go/main/models.ts`）：

```typescript
export namespace main {
    export class Address {
        street: string;
        postcode: string;

        static createFrom(source: any = {}): Address {
            return new Address(source);
        }

        constructor(source: any = {}) {
            if ('string' === typeof source) source = JSON.parse(source);
            this.street = source['street'];
            this.postcode = source['postcode'];
        }
    }

    export class Person {
        name: string;
        age: number;
        address?: Address;

        static createFrom(source: any = {}): Person {
            return new Person(source);
        }

        constructor(source: any = {}) {
            if ('string' === typeof source) source = JSON.parse(source);
            this.name = source['name'];
            this.age = source['age'];
            this.address = this.convertValues(source['address'], Address);
        }

        // ... convertValues 辅助方法
    }
}
```

**完整类型映射表**：

| Go 类型 | TypeScript 类型 | 说明 |
|---------|-----------------|------|
| `string` | `string` | |
| `int`, `int8`..`int64` | `number` | |
| `uint`, `uint8`..`uint64` | `number` | |
| `float32`, `float64` | `number` | |
| `bool` | `boolean` | |
| `[]T` | `T[]` | 切片 → 数组 |
| `map[string]T` | `{[key: string]: T}` | |
| `struct` | `class` | 自动生成 class |
| `*T` | `T \| undefined` | 指针 → 可选 |
| `error`（第二返回值） | Promise reject | 前端通过 catch 捕获 |
| `context.Context` | — | 不参与映射 |

### 2.6 错误处理

当 Go 方法返回 `error` 时，前端的 Promise 会进入 reject 状态：

**Go 端**：

```go
import "errors"

func (a *App) Divide(x, y float64) (float64, error) {
    if y == 0 {
        return 0, errors.New("除数不能为零")
    }
    return x / y, nil
}
```

**前端调用**：

```typescript
import { Divide } from '../wailsjs/go/main/App'

try {
    const result = await Divide(10, 0)
} catch (err) {
    console.error(err) // "除数不能为零"
}
```

### 2.7 绑定多个服务

实际项目中通常会按功能拆分为多个 Go 结构体：

```go
// user_service.go
type UserService struct {
    ctx context.Context
}

func NewUserService() *UserService {
    return &UserService{}
}

func (u *UserService) startup(ctx context.Context) {
    u.ctx = ctx
}

func (u *UserService) GetUser(id int) User {
    // ...
}

// file_service.go
type FileService struct {
    ctx context.Context
}

func NewFileService() *FileService {
    return &FileService{}
}

func (f *FileService) ReadFile(path string) (string, error) {
    // ...
}
```

在 `main.go` 中注册多个绑定：

```go
userService := NewUserService()
fileService := NewFileService()

err := wails.Run(&options.App{
    // ...
    OnStartup: func(ctx context.Context) {
        userService.startup(ctx)
        fileService.startup(ctx)
    },
    Bind: []interface{}{
        userService,
        fileService,
    },
})
```

前端分别导入：

```typescript
import { GetUser } from '../wailsjs/go/main/UserService'
import { ReadFile } from '../wailsjs/go/main/FileService'
```

## 3. 应用生命周期

Wails 提供四个生命周期回调函数，在应用不同阶段触发：

```
应用启动
  │
  ▼
OnStartup(ctx)        ← Go 后端初始化（保存 ctx、连接数据库等）
  │
  ▼
前端加载 index.html
  │
  ▼
OnDomReady(ctx)       ← DOM 准备就绪（可以安全操作窗口、发送初始事件）
  │
  ▼
应用运行中...
  │
  ▼
用户关闭窗口
  │
  ▼
OnBeforeClose(ctx)    ← 关闭前拦截（返回 true 阻止关闭，可弹出确认对话框）
  │
  ▼
OnShutdown(ctx)       ← 应用即将退出（清理资源、关闭连接）
  │
  ▼
应用退出
```

### 3.1 OnStartup — 应用启动

**触发时机**：前端加载之前，Go 后端初始化时调用。

**典型用途**：保存 context、初始化数据库连接、加载配置文件。

```go
func (a *App) startup(ctx context.Context) {
    // ⭐ 保存 context — 后续调用 runtime API（对话框、事件等）都需要它
    a.ctx = ctx

    // 初始化数据库
    db, err := sql.Open("sqlite3", "app.db")
    if err != nil {
        log.Fatal(err)
    }
    a.db = db

    // 加载配置
    a.config = loadConfig()
}
```

> **重要**：`context.Context` 是调用所有 Wails 运行时 API 的前提。必须在 `OnStartup` 中保存它。

### 3.2 OnDomReady — DOM 加载完成

**触发时机**：前端已完成 `index.html` 中所有资源的加载，等同于 JS 中的 `body onload`。

**典型用途**：向前端发送初始化数据、调用需要窗口的 API。

```go
func (a *App) domReady(ctx context.Context) {
    // 此时前端已准备好接收事件
    runtime.EventsEmit(ctx, "app:ready", map[string]interface{}{
        "version": "1.0.0",
        "user":    a.config.LastUser,
    })

    // 可以安全地操作窗口
    runtime.WindowCenter(ctx)
}
```

### 3.3 OnBeforeClose — 关闭前拦截

**触发时机**：用户点击关闭按钮或调用 `runtime.Quit()` 时。

**返回值**：`true` = 阻止关闭；`false` = 允许关闭。

**典型用途**：未保存内容的确认对话框。

```go
func (a *App) beforeClose(ctx context.Context) (prevent bool) {
    if !a.hasUnsavedChanges {
        return false // 没有未保存的内容，允许关闭
    }

    // 弹出确认对话框
    dialog, err := runtime.MessageDialog(ctx, runtime.MessageDialogOptions{
        Type:          runtime.QuestionDialog,
        Title:         "确认退出",
        Message:       "有未保存的更改，确定要退出吗？",
        Buttons:       []string{"保存并退出", "不保存", "取消"},
        DefaultButton: "保存并退出",
        CancelButton:  "取消",
    })
    if err != nil {
        return false
    }

    switch dialog {
    case "保存并退出":
        a.save()
        return false // 保存后允许关闭
    case "不保存":
        return false // 直接关闭
    case "取消":
        return true // 阻止关闭
    }
    return false
}
```

### 3.4 OnShutdown — 应用退出

**触发时机**：应用即将退出（`OnBeforeClose` 允许关闭之后）。

**典型用途**：关闭数据库连接、释放资源、保存状态。

```go
func (a *App) shutdown(ctx context.Context) {
    // 关闭数据库连接
    if a.db != nil {
        a.db.Close()
    }

    // 保存应用状态
    saveConfig(a.config)

    log.Println("应用已退出")
}
```

### 3.5 完整注册示例

```go
func main() {
    app := NewApp()

    err := wails.Run(&options.App{
        Title:         "My App",
        Width:         1024,
        Height:        768,
        OnStartup:     app.startup,     // 启动时
        OnDomReady:    app.domReady,     // DOM 就绪
        OnBeforeClose: app.beforeClose,  // 关闭前拦截
        OnShutdown:    app.shutdown,     // 退出时
        Bind:          []interface{}{app},
    })

    if err != nil {
        log.Fatal(err)
    }
}
```

## 4. 应用配置选项详解

### 4.1 窗口选项（options.App）

`wails.Run()` 接收 `options.App` 结构体，所有应用行为在此处配置。

```go
err := wails.Run(&options.App{
    // ========== 基本窗口属性 ==========
    Title:            "My App",         // 窗口标题
    Width:            1024,             // 初始宽度（默认 1024）
    Height:           768,              // 初始高度（默认 768）
    DisableResize:    false,            // 是否禁止调整窗口大小
    Frameless:        false,            // 无边框模式（去掉标题栏和边框）
    AlwaysOnTop:      false,            // 窗口始终置顶

    // ========== 窗口尺寸约束 ==========
    MinWidth:         400,              // 最小宽度
    MinHeight:        300,              // 最小高度
    MaxWidth:         1920,             // 最大宽度
    MaxHeight:        1080,             // 最大高度

    // ========== 窗口启动状态 ==========
    WindowStartState: options.Normal,   // Normal | Maximised | Minimised | Fullscreen
    StartHidden:      false,            // 启动时隐藏（配合 runtime.WindowShow 使用）
    HideWindowOnClose: false,           // 关闭时隐藏而非退出（常驻托盘场景）

    // ========== 外观 ==========
    BackgroundColour: &options.RGBA{    // 窗口背景色
        R: 27, G: 38, B: 54, A: 255,   // 深蓝色背景
    },

    // ========== 资源服务 ==========
    AssetServer: &assetserver.Options{
        Assets:     assets,             // embed.FS 嵌入的前端资源
        Handler:    nil,                // 自定义资源处理器（http.Handler）
        Middleware: nil,                // 资源请求中间件
    },

    // ========== 生命周期 ==========
    OnStartup:     app.startup,
    OnDomReady:    app.domReady,
    OnBeforeClose: app.beforeClose,
    OnShutdown:    app.shutdown,

    // ========== 绑定 ==========
    Bind: []interface{}{app},

    // ========== 日志 ==========
    LogLevel:           logger.DEBUG,   // 开发环境日志级别
    LogLevelProduction: logger.ERROR,   // 生产环境日志级别

    // ========== 高级选项 ==========
    EnableDefaultContextMenu: false,     // 启用默认右键菜单
    CSSDragProperty: "--wails-draggable",// 无边框窗口拖拽的 CSS 属性名
    CSSDragValue:    "drag",             // CSS 属性值
})
```

### 4.2 窗口启动状态

```go
// 通过 WindowStartState 控制窗口初始状态
WindowStartState: options.Normal,     // 正常显示
WindowStartState: options.Maximised,  // 最大化
WindowStartState: options.Minimised,  // 最小化（Linux 不支持）
WindowStartState: options.Fullscreen, // 全屏
```

### 4.3 无边框窗口拖拽

无边框模式下（`Frameless: true`），需要自定义拖拽区域：

```go
err := wails.Run(&options.App{
    Frameless:       true,
    CSSDragProperty: "--wails-draggable", // 默认值
    CSSDragValue:    "drag",              // 默认值
    // ...
})
```

在前端 CSS 中标记拖拽区域：

```css
/* 标题栏区域 — 可拖拽 */
```

## 5. 项目配置 `wails.json`

`wails.json` 是 Wails 项目的配置文件，使用 JSON5 格式（支持注释）。

### 5.1 完整配置参考

```json5
{
  // 项目名称
  "name": "hello-wails",

  // 输出的二进制文件名
  "outputfilename": "hello-wails",

  // ========== 前端配置 ==========
  // 前端目录（相对项目根目录）
  "frontend:dir": "frontend",

  // 安装前端依赖的命令
  "frontend:install": "npm install",

  // 构建前端的命令（生产环境）
  "frontend:build": "npm run build",

  // 开发模式下构建前端的命令（不指定则回退到 frontend:build）
  "frontend:dev:build": "npm run dev:build",

  // 开发模式下的文件监听命令（在独立进程中运行）
  "frontend:dev:watcher": "npm run dev",

  // 第三方开发服务器 URL（如 Vite）
  // 设为 "auto" 则自动从 Vite 输出中推断
  "frontend:dev:serverUrl": "auto",

  // ========== 资源与构建 ==========
  // 编译后的前端资源目录（通常自动推断，无需设置）
  "assetdir": "",

  // 自动生成的 JS/TS 绑定模块输出目录
  "wailsjsdir": "./frontend",

  // 构建文件目录（默认 "build"）
  "build:dir": "build",

  // 额外触发重载的目录（逗号分隔）
  "reloaddirs": "",

  // 文件变更后重新编译的防抖时间（毫秒）
  "debounceMS": 100,

  // 开发服务器地址
  "devServer": "localhost:34115",

  // 开发模式下传递给应用的参数
  "appargs": "",

  // ========== 构建钩子 ==========
  "preBuildHooks": {
    // 特定平台：${platform} 会被替换为 "GOOS/GOARCH"
    "windows/amd64": "echo Building for Windows amd64",
    // 特定 OS 的所有架构
    "darwin/*": "echo Building for macOS",
    // 所有平台
    "*/*": "echo Building..."
  },
  "postBuildHooks": {
    "*/*": "echo Build complete for ${platform}, binary: ${bin}"
  },

  // ========== 应用元信息 ==========
  "info": {
    "companyName": "My Company",
    "productName": "Hello Wails",
    "productVersion": "1.0.0",
    "copyright": "Copyright 2024 My Company",
    "comments": "Built with Wails"
  }
}
```

### 5.2 关键配置说明

**开发模式的前端配置**：

```
wails dev 启动时执行顺序：
1. frontend:dev:install  (如果指定)  → 安装开发依赖
2. frontend:dev:build    (如果指定)  → 开发模式构建
3. frontend:dev:watcher  (如果指定)  → 启动文件监听（独立进程）
4. frontend:dev:serverUrl              → 连接到开发服务器
```

最常见的 Vite 项目配置：

```json5
{
  "frontend:install": "npm install",
  "frontend:build": "npm run build",
  "frontend:dev:watcher": "npm run dev",
  "frontend:dev:serverUrl": "auto"
}
```

**构建钩子**：
- `preBuildHooks` — 构建前执行，可用于代码生成、版本号注入等
- `postBuildHooks` — 构建后执行，可用于签名、打包、上传等
- 支持 `${platform}`（如 `windows/amd64`）和 `${bin}`（二进制路径）变量替换

## 小结

| 概念 | 要点 |
|------|------|
| CLI | `wails init` 创建项目，`wails dev` 开发，`wails build` 构建 |
| 绑定 | Go 公开方法自动暴露给前端，前端通过 Promise 调用 |
| 类型映射 | Go struct ↔ TS class，自动生成在 `wailsjs/` 目录 |
| 错误处理 | Go 返回 `error` → 前端 Promise reject |
| 生命周期 | `OnStartup` → `OnDomReady` → `OnBeforeClose` → `OnShutdown` |
| Context | 在 `OnStartup` 中保存，是所有运行时 API 调用的钥匙 |
| 配置 | `options.App` 控制窗口行为，`wails.json` 控制项目构建 |
