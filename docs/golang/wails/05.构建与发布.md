# Wails 构建与发布

---

## 1. 构建基础

### 1.1 构建流程概览

```
wails build 执行流程
       │
       ▼
┌─ 1. 执行 preBuildHooks（构建前钩子）
│
├─ 2. 前端构建
│     ├─ frontend:install  → npm install
│     └─ frontend:build    → npm run build
│           输出到 frontend/dist/
│
├─ 3. Go 编译
│     ├─ 将 frontend/dist/ 通过 embed.FS 嵌入
│     ├─ 生成绑定代码
│     ├─ go build -tags desktop,production ...
│     └─ 输出到 build/bin/
│
├─ 4. 平台打包
│     ├─ Windows: 嵌入图标和 manifest
│     ├─ macOS:   生成 .app 包
│     └─ Linux:   生成可执行文件
│
├─ 5. 可选后处理
│     ├─ -nsis:  生成 Windows 安装包
│     ├─ -upx:   UPX 压缩
│     └─ -obfuscated: garble 代码混淆
│
└─ 6. 执行 postBuildHooks（构建后钩子）
```

### 1.2 基本构建命令

```bash
# 构建当前平台的生产版本
wails build

# 查看构建命令但不实际执行（调试用）
wails build -dryrun
```

构建输出示例：

```
Wails CLI v2.10.1

Building target: windows/amd64
------------------------------

  - Compiling frontend: Done.
  - Generating bindings: Done.
  - Compiling application: Done.
  - Processing application icon: Done.
  - Creating .exe: Done.

Built 'C:\projects\myapp\build\bin\myapp.exe' in 12.3s.
```

## 2. wails build 全部选项

### 2.1 常用选项速查

```bash
wails build [选项]
```

| 选项 | 说明 | 示例 |
|------|------|------|
| `-clean` | 清理 `build/bin` 后重新构建 | `wails build -clean` |
| `-platform` | 指定目标平台 | `wails build -platform windows/amd64` |
| `-o` | 自定义输出文件名 | `wails build -o myapp-v1.0.exe` |
| `-ldflags` | 传递给 Go 链接器的参数 | `wails build -ldflags "-s -w"` |
| `-debug` | 保留调试信息 | `wails build -debug` |
| `-devtools` | 生产版本中启用 DevTools | `wails build -devtools` |
| `-nsis` | 生成 Windows NSIS 安装程序 | `wails build -nsis` |
| `-upx` | 使用 UPX 压缩二进制 | `wails build -upx` |
| `-upxflags` | 传递给 UPX 的参数 | `wails build -upx -upxflags "--best"` |
| `-obfuscated` | 使用 garble 混淆代码 | `wails build -obfuscated` |
| `-garbleargs` | 传递给 garble 的参数 | `wails build -obfuscated -garbleargs "-literals"` |
| `-trimpath` | 去除二进制中的路径信息 | `wails build -trimpath` |
| `-race` | 启用竞态检测器 | `wails build -race` |
| `-tags` | 传递给 Go 编译器的构建标签 | `wails build -tags "sqlite_fts5"` |
| `-s` | 跳过前端构建 | `wails build -s` |
| `-skipbindings` | 跳过绑定代码生成 | `wails build -skipbindings` |
| `-m` | 跳过 `go mod tidy` | `wails build -m` |
| `-f` | 强制构建 | `wails build -f` |
| `-nopackage` | 不打包（不生成 .app 等） | `wails build -nopackage` |
| `-webview2` | WebView2 安装策略 | `wails build -webview2 embed` |
| `-windowsconsole` | 保留 Windows 控制台窗口 | `wails build -windowsconsole` |
| `-dryrun` | 打印命令但不执行 | `wails build -dryrun` |
| `-v` | 详细输出 | `wails build -v` |

### 2.2 常见构建场景

**场景一：标准生产版本**

```bash
wails build -clean -trimpath
```

```
  - Cleaning build/bin directory: Done.
  - Installing frontend dependencies: Done.
  - Compiling frontend: Done.
  - Generating bindings: Done.
  - Compiling application: Done.
Built 'build/bin/myapp.exe' in 15.2s.
```

**场景二：调试版本（可用 F12 开启 DevTools）**

```bash
wails build -debug -devtools
```

```
  - Compiling application (Debug Mode): Done.
Built 'build/bin/myapp.exe' in 18.7s.
```

**场景三：最小体积构建**

```bash
wails build -clean -trimpath -ldflags "-s -w" -upx -upxflags "--best"
```

`-s -w` 去除符号表和调试信息，`-upx --best` 最大压缩：

```
构建前:  myapp.exe  →  28.5 MB
去符号:  myapp.exe  →  21.3 MB（-s -w）
UPX:    myapp.exe  →   8.1 MB（--best）
```

**场景四：代码混淆版本（防止反编译）**

```bash
wails build -obfuscated -garbleargs "-literals -tiny -seed=random"
```

| garble 参数 | 说明 |
|-------------|------|
| `-literals` | 混淆字符串字面量 |
| `-tiny` | 移除额外信息以减小体积 |
| `-seed=random` | 使用随机种子（每次构建不同） |
| `-seed=myseed` | 固定种子（可复现构建） |

**场景五：带控制台的 Windows 调试版**

```bash
wails build -windowsconsole -debug
```

构建产物运行时会弹出控制台窗口，可以看到 `fmt.Println` 的输出，便于调试。

## 3. 版本信息注入

### 3.1 定义版本变量

```go
// version.go
package main

// 这些变量的值由构建时的 ldflags 注入
// 如果不注入，则使用这里的默认值
var (
    AppVersion = "dev"
    BuildTime  = "unknown"
    GitCommit  = "unknown"
    GoVersion  = "unknown"
)
```

### 3.2 暴露给前端

```go
type VersionInfo struct {
    AppVersion string `json:"appVersion"`
    BuildTime  string `json:"buildTime"`
    GitCommit  string `json:"gitCommit"`
    GoVersion  string `json:"goVersion"`
    OS         string `json:"os"`
    Arch       string `json:"arch"`
}

func (a *App) GetVersionInfo() VersionInfo {
    info := VersionInfo{
        AppVersion: AppVersion,
        BuildTime:  BuildTime,
        GitCommit:  GitCommit,
        GoVersion:  GoVersion,
        OS:         goruntime.GOOS,
        Arch:       goruntime.GOARCH,
    }
    fmt.Printf("版本信息: v%s (built %s, commit %s)\n",
        info.AppVersion, info.BuildTime, info.GitCommit)
    // 开发时 → 输出: 版本信息: vdev (built unknown, commit unknown)
    // 构建后 → 输出: 版本信息: v1.2.0 (built 2026-03-24T10:00:00, commit a1b2c3d)
    return info
}
```

### 3.3 构建时注入

**手动构建**：

```bash
wails build -ldflags "\
  -X 'main.AppVersion=1.2.0' \
  -X 'main.BuildTime=$(date -u +%Y-%m-%dT%H:%M:%S)' \
  -X 'main.GitCommit=$(git rev-parse --short HEAD)' \
  -X 'main.GoVersion=$(go version | cut -d\" \" -f3)'"
```

**使用脚本**（`scripts/build.sh`）：

```bash
#!/bin/bash
set -e

VERSION=${1:-"0.0.0-dev"}
BUILD_TIME=$(date -u +"%Y-%m-%dT%H:%M:%SZ")
GIT_COMMIT=$(git rev-parse --short HEAD 2>/dev/null || echo "unknown")
GO_VERSION=$(go version | awk '{print $3}')

echo "========================================="
echo "构建版本: $VERSION"
echo "构建时间: $BUILD_TIME"
echo "Git 提交: $GIT_COMMIT"
echo "Go 版本:  $GO_VERSION"
echo "========================================="
# 输出:
# =========================================
# 构建版本: 1.2.0
# 构建时间: 2026-03-24T10:00:00Z
# Git 提交: a1b2c3d
# Go 版本:  go1.22.1
# =========================================

LDFLAGS="-X 'main.AppVersion=$VERSION'"
LDFLAGS="$LDFLAGS -X 'main.BuildTime=$BUILD_TIME'"
LDFLAGS="$LDFLAGS -X 'main.GitCommit=$GIT_COMMIT'"
LDFLAGS="$LDFLAGS -X 'main.GoVersion=$GO_VERSION'"
LDFLAGS="$LDFLAGS -s -w"  # 去除符号表

wails build -clean -trimpath -ldflags "$LDFLAGS"

echo "构建完成！"
ls -lh build/bin/
# 输出:
# 构建完成！
# -rwxr-xr-x 1 user user 21M Mar 24 10:01 myapp.exe
```

使用方法：

```bash
# 开发版
bash scripts/build.sh

# 指定版本号
bash scripts/build.sh 1.2.0
```

## 4. 应用图标与元信息

### 4.1 应用图标

Wails 项目创建时会在 `build/appicon.png` 生成默认图标。替换此文件即可自定义图标。

**图标要求**：

| 平台 | 文件 | 要求 |
|------|------|------|
| 源文件 | `build/appicon.png` | PNG 格式，推荐 1024x1024 |
| Windows | 自动生成 `build/windows/icon.ico` | 构建时从 PNG 转换 |
| macOS | 自动生成 `.icns` | 构建时从 PNG 转换 |
| Linux | 嵌入二进制 | 使用原始 PNG |

替换图标后重新构建即可：

```bash
# 替换图标
cp /path/to/my-icon.png build/appicon.png

# 重新构建（-clean 确保图标更新）
wails build -clean
```

```
  - Processing application icon: Done.    ← 图标已更新
  - Creating .exe: Done.
```

## 5. 构建钩子

### 5.1 配置构建钩子

在 `wails.json` 中设置 `preBuildHooks` 和 `postBuildHooks`：

```json
{
    "preBuildHooks": {
        "windows/amd64": "echo 构建 Windows amd64 版本",
        "darwin/*": "echo 构建 macOS 版本",
        "*/*": "echo 开始构建 ${platform}"
    },
    "postBuildHooks": {
        "windows/amd64": "echo Windows 构建完成，产物: ${bin}",
        "*/*": "echo 构建完成: ${platform} → ${bin}"
    }
}
```

**变量替换**：

| 变量 | 说明 | 示例值 |
|------|------|--------|
| `${platform}` | 目标平台 | `windows/amd64` |
| `${bin}` | 输出二进制路径（仅 postBuild） | `build/bin/myapp.exe` |

**执行顺序**（以 `windows/amd64` 为例）：

```
preBuildHooks:
  1. "windows/amd64" 钩子        → echo 构建 Windows amd64 版本
  2. "*/*" 钩子                  → echo 开始构建 windows/amd64

（执行构建）

postBuildHooks:
  1. "windows/amd64" 钩子        → echo Windows 构建完成，产物: build/bin/myapp.exe
  2. "*/*" 钩子                  → echo 构建完成: windows/amd64 → build/bin/myapp.exe
```

### 5.2 实际应用：构建前自动生成版本文件

```json
{
    "preBuildHooks": {
        "*/*": "go run scripts/gen_version.go"
    }
}
```

`scripts/gen_version.go`：

```go
//go:build ignore

package main

import (
    "fmt"
    "os"
    "os/exec"
    "strings"
    "time"
)

func main() {
    commit := "unknown"
    out, err := exec.Command("git", "rev-parse", "--short", "HEAD").Output()
    if err == nil {
        commit = strings.TrimSpace(string(out))
    }

    content := fmt.Sprintf(`package main

// 由构建钩子自动生成，请勿手动修改
const (
    generatedBuildTime  = "%s"
    generatedGitCommit  = "%s"
)
`, time.Now().UTC().Format("2006-01-02T15:04:05Z"), commit)

    os.WriteFile("build_info_generated.go", []byte(content), 0644)
    fmt.Println("已生成 build_info_generated.go")
    // 输出: 已生成 build_info_generated.go
}
```

## 6. 跨平台打包

### 6.1 Windows 打包

#### NSIS 安装程序

```bash
wails build -clean -nsis
```

```
  - Compiling application: Done.
  - Creating .exe: Done.
  - Creating NSIS installer: Done.

Built:
  - build/bin/MyApp.exe                    # 可执行文件（21.3 MB）
  - build/bin/MyApp-amd64-installer.exe    # NSIS 安装程序（22.1 MB）
```

NSIS 安装程序的特性：
- 安装向导界面，显示 `wails.json` 中的产品信息
- 可选安装路径
- 自动创建桌面快捷方式
- 自动创建开始菜单条目
- 带卸载程序

#### 自定义 NSIS 脚本

安装脚本在 `build/windows/installer/project.nsi`，可以自定义安装行为。常见修改：

```nsi
;; build/windows/installer/project.nsi（部分）

;; 自定义安装目录
InstallDir "$PROGRAMFILES64\${INFO_PRODUCTNAME}"

;; 安装后自动启动
Section
    ; ... 默认安装内容 ...

    ; 安装完成后启动应用
    Exec "$INSTDIR\${PRODUCT_EXECUTABLE}"
SectionEnd
```

#### 代码签名与公证

```bash
#!/bin/bash
APP_CERTIFICATE="Developer ID Application: YOUR NAME (TEAM_ID)"
APP_NAME="MyApp"

# 1. 构建
wails build -platform darwin/universal -clean

# 2. 签名
codesign --timestamp --options=runtime \
    -s "$APP_CERTIFICATE" \
    -v --entitlements ./build/darwin/entitlements.plist \
    "./build/bin/$APP_NAME.app"

echo "签名完成"
# 输出: 签名完成

# 3. 公证（Notarization）
xcrun notarytool submit "./build/bin/$APP_NAME.app" \
    --apple-id "your@email.com" \
    --team-id "TEAM_ID" \
    --password "app-specific-password" \
    --wait

echo "公证完成"
# 输出: 公证完成

# 4. 装订公证票据
xcrun stapler staple "./build/bin/$APP_NAME.app"

echo "装订完成"
# 输出: 装订完成
```

**entitlements.plist** — 签名权限文件：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN"
  "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>com.apple.security.app-sandbox</key>
    <false/>
    <key>com.apple.security.network.client</key>
    <true/>
    <key>com.apple.security.files.user-selected.read-write</key>
    <true/>
</dict>
</plist>
```

### 6.3 Linux 打包

#### 基本构建

```bash
wails build -clean -platform linux/amd64
```

```
Built: build/bin/myapp (24.6 MB)
```

#### 创建 AppImage

使用 `appimagetool`：

```bash
# 1. 创建 AppDir 结构
mkdir -p MyApp.AppDir/usr/bin
mkdir -p MyApp.AppDir/usr/share/icons/hicolor/256x256/apps

# 2. 复制文件
cp build/bin/myapp MyApp.AppDir/usr/bin/
cp build/appicon.png MyApp.AppDir/usr/share/icons/hicolor/256x256/apps/myapp.png
cp build/appicon.png MyApp.AppDir/myapp.png

# 3. 创建 .desktop 文件
cat > MyApp.AppDir/myapp.desktop << 'EOF'
[Desktop Entry]
Name=MyApp
Exec=myapp
Icon=myapp
Type=Application
Categories=Utility;
Comment=基于 Wails 构建的跨平台应用
EOF

# 4. 创建 AppRun
cat > MyApp.AppDir/AppRun << 'APPRUN'
#!/bin/bash
SELF=$(readlink -f "$0")
HERE=${SELF%/*}
exec "${HERE}/usr/bin/myapp" "$@"
APPRUN
chmod +x MyApp.AppDir/AppRun

# 5. 生成 AppImage
appimagetool MyApp.AppDir MyApp-x86_64.AppImage
```

```
Generating AppImage...
  Embedding runtime: Done.
  Creating squashfs: Done.

输出: MyApp-x86_64.AppImage (25.1 MB)
```

## 7. 手动构建流程

当需要完全控制构建过程时，可以绕过 `wails build` 手动执行每一步。

### 7.1 步骤分解

```bash
# 步骤 1: 安装前端依赖
cd frontend && npm install && cd ..

# 步骤 2: 构建前端
cd frontend && npm run build && cd ..

# 步骤 3: 生成绑定
wails generate module

# 步骤 4: Go 编译（生产模式）
go build -tags desktop,production -trimpath -ldflags "-s -w" -o build/bin/myapp.exe

# 步骤 5: (可选) UPX 压缩
upx --best build/bin/myapp.exe
```

### 7.2 构建标签说明

| 标签 | 用途 |
|------|------|
| `desktop` | 桌面应用构建（始终需要） |
| `production` | 生产模式，使用 `embed.FS` 加载资源 |
| `dev` | 开发模式，从开发服务器加载资源 |
| `debug` | 启用调试功能 |

```bash
# 开发调试构建
go build -tags "desktop,dev" -gcflags "all=-N -l" -o build/bin/myapp-debug.exe

# 生产构建
go build -tags "desktop,production" -trimpath -ldflags "-s -w" -o build/bin/myapp.exe
```

### 7.3 使用 Makefile 管理

```makefile
# Makefile
APP_NAME := myapp
VERSION  := $(shell git describe --tags --always --dirty 2>/dev/null || echo "dev")
BUILD_TIME := $(shell date -u +"%Y-%m-%dT%H:%M:%SZ")
GIT_COMMIT := $(shell git rev-parse --short HEAD 2>/dev/null || echo "unknown")

LDFLAGS := -s -w
LDFLAGS += -X 'main.AppVersion=$(VERSION)'
LDFLAGS += -X 'main.BuildTime=$(BUILD_TIME)'
LDFLAGS += -X 'main.GitCommit=$(GIT_COMMIT)'

.PHONY: dev build build-windows build-mac build-linux clean

# 开发模式
dev:
	wails dev

# 当前平台构建
build:
	wails build -clean -trimpath -ldflags "$(LDFLAGS)"
	@echo "✓ 构建完成: build/bin/"
	@ls -lh build/bin/

# Windows 构建（含安装包）
build-windows:
	wails build -clean -platform windows/amd64 -nsis -trimpath -ldflags "$(LDFLAGS)"
	@echo "✓ Windows 构建完成"
	@ls -lh build/bin/

# macOS 构建
build-mac:
	wails build -clean -platform darwin/universal -trimpath -ldflags "$(LDFLAGS)"
	@echo "✓ macOS 构建完成"
	@ls -lh build/bin/

# Linux 构建
build-linux:
	wails build -clean -platform linux/amd64 -trimpath -ldflags "$(LDFLAGS)"
	@echo "✓ Linux 构建完成"
	@ls -lh build/bin/

# 清理
clean:
	rm -rf build/bin/*
	@echo "✓ 已清理 build/bin/"
```

使用：

```bash
make build
# 输出:
# ✓ 构建完成: build/bin/
# -rwxr-xr-x 1 user user 21M Mar 24 10:30 myapp.exe

make build VERSION=1.2.0
# 构建时 AppVersion 被设为 1.2.0
```

## 8. 自动更新

### 8.1 版本检查

```go
package main

import (
    "context"
    "encoding/json"
    "fmt"
    "net/http"
    "time"

    "github.com/wailsapp/wails/v2/pkg/runtime"
)

type ReleaseInfo struct {
    Version     string `json:"version"`
    DownloadURL string `json:"downloadUrl"`
    ReleaseNote string `json:"releaseNote"`
    Date        string `json:"date"`
}

type UpdateService struct {
    ctx       context.Context
    updateURL string // 版本检查 API 地址
}

func NewUpdateService(url string) *UpdateService {
    return &UpdateService{updateURL: url}
}

func (u *UpdateService) startup(ctx context.Context) {
    u.ctx = ctx

    // 启动后延迟检查更新（不阻塞启动）
    go func() {
        time.Sleep(3 * time.Second)
        u.CheckForUpdates(true) // 静默检查
    }()
}

// 检查更新
func (u *UpdateService) CheckForUpdates(silent bool) (*ReleaseInfo, error) {
    fmt.Println("正在检查更新...")
    // 输出: 正在检查更新...

    client := &http.Client{Timeout: 10 * time.Second}
    resp, err := client.Get(u.updateURL)
    if err != nil {
        fmt.Printf("检查更新失败: %v\n", err)
        // 输出: 检查更新失败: Get "...": dial tcp: ...
        if !silent {
            runtime.MessageDialog(u.ctx, runtime.MessageDialogOptions{
                Type:    runtime.ErrorDialog,
                Title:   "检查更新失败",
                Message: "无法连接到更新服务器，请检查网络连接。",
            })
        }
        return nil, err
    }
    defer resp.Body.Close()

    var release ReleaseInfo
    if err := json.NewDecoder(resp.Body).Decode(&release); err != nil {
        return nil, err
    }

    fmt.Printf("最新版本: %s, 当前版本: %s\n", release.Version, AppVersion)
    // 输出: 最新版本: 1.3.0, 当前版本: 1.2.0

    // 比较版本号
    if release.Version == AppVersion {
        fmt.Println("已是最新版本")
        // 输出: 已是最新版本
        if !silent {
            runtime.MessageDialog(u.ctx, runtime.MessageDialogOptions{
                Type:    runtime.InfoDialog,
                Title:   "检查更新",
                Message: fmt.Sprintf("当前已是最新版本 v%s", AppVersion),
            })
        }
        return nil, nil
    }

    // 有新版本
    fmt.Printf("发现新版本: %s → %s\n", AppVersion, release.Version)
    // 输出: 发现新版本: 1.2.0 → 1.3.0

    if !silent {
        result, _ := runtime.MessageDialog(u.ctx, runtime.MessageDialogOptions{
            Type:    runtime.QuestionDialog,
            Title:   "发现新版本",
            Message: fmt.Sprintf("新版本 v%s 已发布！\n\n%s\n\n是否前往下载？", release.Version, release.ReleaseNote),
            Buttons: []string{"前往下载", "稍后提醒"},
        })

        if result == "前往下载" {
            runtime.BrowserOpenURL(u.ctx, release.DownloadURL)
            fmt.Println("已打开下载页面")
            // 输出: 已打开下载页面
        }
    }

    return &release, nil
}
```

### 8.2 更新检查 API 格式

后端 API 只需返回一个 JSON：

```json
{
    "version": "1.3.0",
    "downloadUrl": "https://github.com/myorg/myapp/releases/tag/v1.3.0",
    "releaseNote": "新增暗色模式支持，修复了文件保存的 Bug",
    "date": "2026-03-20"
}
```

也可以直接使用 GitHub Releases API：

```go
func NewUpdateService() *UpdateService {
    return &UpdateService{
        updateURL: "https://api.github.com/repos/myorg/myapp/releases/latest",
    }
}

// GitHub Releases 格式解析
type GitHubRelease struct {
    TagName string `json:"tag_name"` // "v1.3.0"
    HTMLURL string `json:"html_url"` // Release 页面链接
    Body    string `json:"body"`     // Release Notes
}

func (u *UpdateService) CheckGitHubRelease() (*ReleaseInfo, error) {
    // ...请求 GitHub API...

    var ghRelease GitHubRelease
    json.NewDecoder(resp.Body).Decode(&ghRelease)

    version := strings.TrimPrefix(ghRelease.TagName, "v")
    fmt.Printf("GitHub 最新版本: %s\n", version)
    // 输出: GitHub 最新版本: 1.3.0

    return &ReleaseInfo{
        Version:     version,
        DownloadURL: ghRelease.HTMLURL,
        ReleaseNote: ghRelease.Body,
    }, nil
}
```

## 9. CI/CD 自动构建与发布

### 9.1 GitHub Actions 完整工作流

```yaml
# .github/workflows/release.yml
name: Release

on:
  push:
    tags:
      - 'v*'  # 推送 v 开头的 tag 时触发

env:
  NODE_OPTIONS: "--max-old-space-size=4096"
  APP_NAME: "MyApp"

jobs:
  # ========== 多平台构建 ==========
  build:
    strategy:
      fail-fast: false
      matrix:
        include:
          - platform: 'linux/amd64'
            os: 'ubuntu-latest'
            output: 'MyApp'
          - platform: 'windows/amd64'
            os: 'windows-latest'
            output: 'MyApp.exe'
          - platform: 'darwin/universal'
            os: 'macos-latest'
            output: 'MyApp.app'

    runs-on: ${{ matrix.os }}

    steps:
      - name: Checkout
        uses: actions/checkout@v4
        with:
          submodules: recursive

      - name: Setup Go
        uses: actions/setup-go@v5
        with:
          go-version: '1.22'

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'

      # Linux 依赖
      - name: Install Linux dependencies
        if: runner.os == 'Linux'
        run: |
          sudo apt-get update
          sudo apt-get install -y libgtk-3-dev libwebkit2gtk-4.0-dev
          echo "Linux 依赖安装完成"

      - name: Install Wails CLI
        run: go install github.com/wailsapp/wails/v2/cmd/wails@latest

      # 提取版本号
      - name: Get version
        id: version
        run: echo "version=${GITHUB_REF#refs/tags/v}" >> $GITHUB_OUTPUT
        shell: bash

      # 构建
      - name: Build
        run: |
          wails build -clean -trimpath \
            -ldflags "-s -w -X 'main.AppVersion=${{ steps.version.outputs.version }}' -X 'main.BuildTime=$(date -u +%Y-%m-%dT%H:%M:%SZ)' -X 'main.GitCommit=${{ github.sha }}'"
        shell: bash

      # Windows: 额外生成 NSIS 安装包
      - name: Build NSIS Installer
        if: runner.os == 'Windows'
        run: wails build -nsis -skipbindings -s
        shell: bash

      # 打包产物
      - name: Package (Linux)
        if: runner.os == 'Linux'
        run: |
          cd build/bin
          tar -czf MyApp-linux-amd64.tar.gz MyApp
          echo "打包完成: MyApp-linux-amd64.tar.gz"

      - name: Package (macOS)
        if: runner.os == 'macOS'
        run: |
          cd build/bin
          zip -r MyApp-macos-universal.zip MyApp.app
          echo "打包完成: MyApp-macos-universal.zip"

      # 上传构建产物
      - name: Upload artifacts
        uses: actions/upload-artifact@v4
        with:
          name: build-${{ matrix.os }}
          path: |
            build/bin/*.tar.gz
            build/bin/*.zip
            build/bin/*.exe

  # ========== 创建 Release ==========
  release:
    needs: build
    runs-on: ubuntu-latest

    steps:
      - name: Download all artifacts
        uses: actions/download-artifact@v4
        with:
          path: artifacts
          merge-multiple: true

      - name: List artifacts
        run: |
          find artifacts -type f
          echo "---"
          # 输出:
          # artifacts/MyApp-linux-amd64.tar.gz
          # artifacts/MyApp.exe
          # artifacts/MyApp-amd64-installer.exe
          # artifacts/MyApp-macos-universal.zip

      - name: Create GitHub Release
        uses: softprops/action-gh-release@v2
        with:
          files: artifacts/**/*
          generate_release_notes: true
          draft: false
          prerelease: ${{ contains(github.ref, '-rc') || contains(github.ref, '-beta') }}
```

### 9.2 触发构建

```bash
# 创建版本 tag
git tag v1.2.0

# 推送 tag 触发 CI
git push origin v1.2.0
```

GitHub Actions 会自动：
1. 在 Linux、Windows、macOS 三个平台上并行构建
2. Windows 额外生成 NSIS 安装程序
3. Linux 打包为 `.tar.gz`，macOS 打包为 `.zip`
4. 创建 GitHub Release 并上传所有构建产物
5. 自动生成 Release Notes（基于 commit 历史）

### 9.3 版本号管理策略

推荐使用 [语义化版本](https://semver.org/lang/zh-CN/)：

```
v<主版本>.<次版本>.<修订号>[-预发布标识]

v1.0.0       正式版
v1.1.0       新功能
v1.1.1       Bug 修复
v2.0.0       不兼容的重大变更
v1.2.0-beta  预发布版
v1.2.0-rc.1  发布候选
```

完整的发布流程：

```bash
# 1. 更新 wails.json 中的版本号
# "productVersion": "1.2.0"

# 2. 提交版本变更
git add wails.json
git commit -m "chore: bump version to 1.2.0"

# 3. 创建 tag
git tag -a v1.2.0 -m "Release v1.2.0: 新增暗色模式，修复文件保存 Bug"

# 4. 推送代码和 tag
git push origin main
git push origin v1.2.0

# 5. CI 自动构建并发布到 GitHub Releases
```

## 小结

| 主题 | 核心要点 |
|------|----------|
| 基本构建 | `wails build` 一键构建，`-clean` 清理缓存 |
| 构建选项 | `-trimpath -ldflags "-s -w"` 最小体积，`-upx` 压缩，`-obfuscated` 混淆 |
| 版本注入 | `-ldflags "-X main.Version=1.0"` 编译时注入变量 |
| 应用图标 | 替换 `build/appicon.png`，构建时自动转换各平台格式 |
| 元信息 | `wails.json` 的 `info` 字段，影响安装包和系统属性显示 |
| 构建钩子 | `preBuildHooks`/`postBuildHooks`，支持 `${platform}` 和 `${bin}` 变量 |
| Windows 打包 | `-nsis` 生成安装程序，`-webview2 embed` 离线部署 |
| macOS 打包 | `darwin/universal` 通用二进制，`create-dmg` 生成安装镜像，签名 + 公证 |
| Linux 打包 | AppImage / .deb / .rpm，需手动创建目录结构 |
| 手动构建 | `npm run build` → `wails generate module` → `go build -tags desktop,production` |
| 自动更新 | 请求版本 API 比对版本号，引导用户下载 |
| CI/CD | GitHub Actions 矩阵构建三平台，tag 触发自动发布 |
| 版本管理 | 语义化版本 + Git Tag + ldflags 注入 |
