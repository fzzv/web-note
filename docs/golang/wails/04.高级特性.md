## 1. 自定义协议与深度链接

自定义协议（Custom Protocol）允许其他应用或浏览器通过 `myapp://` 这样的 URI 打开你的应用，并传递参数。

### 1.1 配置自定义协议

在 `wails.json` 中注册：

```json
{
  "info": {
    "protocols": [
      {
        "scheme": "myapp",
        "description": "My App Protocol",
        "role": "Editor"
      }
    ]
  }
}
```

| 字段 | 说明 | 平台 |
|------|------|------|
| `scheme` | 协议名称，如 `myapp`（不含 `://`） | 全平台 |
| `description` | 协议描述 | Windows |
| `role` | 应用角色：`Editor`、`Viewer`、`Shell`、`None` | macOS |

### 1.2 处理深度链接（macOS）

macOS 通过 `OnUrlOpen` 回调处理：

```go
package main

import (
    "fmt"
    "github.com/wailsapp/wails/v2/pkg/options/mac"
    "github.com/wailsapp/wails/v2/pkg/runtime"
)

type App struct {
    ctx context.Context
}

func main() {
    app := &App{}

    err := wails.Run(&options.App{
        Title:  "My App",
        Width:  1024,
        Height: 768,
        Mac: &mac.Options{
            OnUrlOpen: func(url string) {
                fmt.Printf("收到深度链接: %s\n", url)
                // 用户点击 myapp://open?file=test.txt 时
                // 输出: 收到深度链接: myapp://open?file=test.txt

                // 通过事件通知前端
                runtime.EventsEmit(app.ctx, "deeplink:open", url)
            },
        },
        OnStartup: app.startup,
        Bind:      []interface{}{app},
    })
    if err != nil {
        log.Fatal(err)
    }
}

func (a *App) startup(ctx context.Context) {
    a.ctx = ctx
}
```

### 1.3 处理深度链接（Windows / Linux）

Windows 和 Linux 通过**启动参数**传递 URL，可结合单实例锁定来处理：

```go
import "github.com/wailsapp/wails/v2/pkg/options"

func main() {
    app := &App{}

    // 检查启动参数中是否有深度链接
    if len(os.Args) > 1 {
        url := os.Args[1]
        fmt.Printf("启动参数中的链接: %s\n", url)
        // 输出: 启动参数中的链接: myapp://open?file=test.txt
    }

    err := wails.Run(&options.App{
        Title: "My App",
        SingleInstanceLock: &options.SingleInstanceLock{
            UniqueId: "my-app-unique-id",
            OnSecondInstanceLaunch: func(data options.SecondInstanceData) {
                // 第二个实例启动时（可能携带深度链接参数）
                for _, arg := range data.Args {
                    if strings.HasPrefix(arg, "myapp://") {
                        fmt.Printf("第二实例传入链接: %s\n", arg)
                        // 输出: 第二实例传入链接: myapp://open?file=test.txt
                        runtime.EventsEmit(app.ctx, "deeplink:open", arg)
                    }
                }
                // 显示已存在的窗口
                runtime.WindowUnminimise(app.ctx)
                runtime.Show(app.ctx)
            },
        },
        OnStartup: app.startup,
        Bind:      []interface{}{app},
    })
}
```

### 1.4 前端解析深度链接

```typescript
import { EventsOn } from '../wailsjs/runtime/runtime'

EventsOn("deeplink:open", (url: string) => {
    console.log("收到深度链接:", url)
    // 输出: 收到深度链接: myapp://open?file=test.txt

    // 解析 URL
    const parsed = new URL(url)
    console.log("协议:", parsed.protocol)   // 输出: 协议: myapp:
    console.log("路径:", parsed.pathname)    // 输出: 路径: open
    console.log("参数:", parsed.searchParams.get("file"))  // 输出: 参数: test.txt

    // 根据路径执行不同操作
    switch (parsed.pathname.replace(/^\/\//, '')) {
        case "open":
            openFile(parsed.searchParams.get("file") || "")
            break
        case "settings":
            router.push("/settings")
            break
    }
})
```

## 2. 文件关联

文件关联让用户双击特定扩展名的文件时，自动用你的应用打开。

### 2.1 配置文件关联

在 `wails.json` 中配置：

```json
{
  "info": {
    "fileAssociations": [
      {
        "ext": "mnote",
        "name": "MyNote File",
        "description": "MyNote 笔记文件",
        "iconName": "fileIcon",
        "role": "Editor"
      },
      {
        "ext": "mnote-template",
        "name": "MyNote Template",
        "description": "MyNote 模板文件",
        "iconName": "templateIcon",
        "role": "Viewer"
      }
    ]
  }
}
```

| 字段 | 说明 |
|------|------|
| `ext` | 文件扩展名（不含前导 `.`） |
| `name` | 文件类型名称 |
| `description` | 文件类型描述（Windows 资源管理器中显示） |
| `iconName` | 图标名称（不含扩展名，图标放在 `build/` 目录下） |
| `role` | macOS 专用：`Editor`（可读写）、`Viewer`（只读） |

### 2.2 处理文件打开事件

**macOS** — 通过 `OnFileOpen` 回调：

```go
Mac: &mac.Options{
    OnFileOpen: func(filePath string) {
        fmt.Printf("通过文件关联打开: %s\n", filePath)
        // 双击 example.mnote 时
        // 输出: 通过文件关联打开: /Users/xxx/Documents/example.mnote

        runtime.EventsEmit(app.ctx, "file:openExternal", filePath)
    },
},
```

**Windows / Linux** — 通过启动参数：

```go
func (a *App) startup(ctx context.Context) {
    a.ctx = ctx

    // 检查启动参数中是否有关联文件
    if len(os.Args) > 1 {
        filePath := os.Args[1]
        if strings.HasSuffix(filePath, ".mnote") {
            fmt.Printf("通过文件关联打开: %s\n", filePath)
            // 输出: 通过文件关联打开: C:\Users\docs\example.mnote

            // DOM 可能还没准备好，延迟发送事件
            go func() {
                time.Sleep(500 * time.Millisecond)
                runtime.EventsEmit(ctx, "file:openExternal", filePath)
            }()
        }
    }
}
```

## 3. 枚举绑定（EnumBind）

除了绑定结构体方法，Wails 还支持将 Go 枚举类型映射到前端 TypeScript 枚举。

### 3.1 定义枚举

```go
package main

// 定义枚举类型
type Priority string

const (
    PriorityLow    Priority = "low"
    PriorityMedium Priority = "medium"
    PriorityHigh   Priority = "high"
    PriorityUrgent Priority = "urgent"
)

// 枚举值列表 — Wails 要求的特定格式
var AllPriorities = []struct {
    Value  Priority
    TSName string   // TypeScript 中的常量名
}{
    {PriorityLow, "LOW"},
    {PriorityMedium, "MEDIUM"},
    {PriorityHigh, "HIGH"},
    {PriorityUrgent, "URGENT"},
}

// 状态枚举
type TaskStatus string

const (
    StatusPending    TaskStatus = "pending"
    StatusInProgress TaskStatus = "in_progress"
    StatusDone       TaskStatus = "done"
    StatusCancelled  TaskStatus = "cancelled"
)

var AllTaskStatuses = []struct {
    Value  TaskStatus
    TSName string
}{
    {StatusPending, "PENDING"},
    {StatusInProgress, "IN_PROGRESS"},
    {StatusDone, "DONE"},
    {StatusCancelled, "CANCELLED"},
}
```

### 3.2 注册枚举绑定

```go
err := wails.Run(&options.App{
    // ...
    Bind: []interface{}{app},
    EnumBind: []interface{}{
        AllPriorities,    // 注册优先级枚举
        AllTaskStatuses,  // 注册状态枚举
    },
})
```

### 3.3 自动生成的 TypeScript 枚举

Wails 会在 `wailsjs/go/models.ts` 中生成：

```typescript
export namespace main {
    export enum Priority {
        LOW = "low",
        MEDIUM = "medium",
        HIGH = "high",
        URGENT = "urgent",
    }

    export enum TaskStatus {
        PENDING = "pending",
        IN_PROGRESS = "in_progress",
        DONE = "done",
        CANCELLED = "cancelled",
    }
}
```

### 3.4 在前端中使用

```typescript
import { main } from '../wailsjs/go/models'

// 使用枚举值
const task = {
    title: "完成文档",
    priority: main.Priority.HIGH,
    status: main.TaskStatus.PENDING,
}

console.log(task.priority)  // 输出: high
console.log(task.status)    // 输出: pending

// 枚举比较
if (task.priority === main.Priority.URGENT) {
    console.log("紧急任务！")
}

// 遍历枚举
Object.values(main.Priority).forEach(p => {
    console.log(p)
})
// 输出:
// low
// medium
// high
// urgent
```

## 4. 平台特定功能

### 4.1 Windows 特性

#### 窗口背景效果

```go
import "github.com/wailsapp/wails/v2/pkg/options/windows"

Windows: &windows.Options{
    // WebView 背景透明（需要配合 BackdropType 使用）
    WebviewIsTransparent: true,

    // 窗口半透明
    WindowIsTranslucent: true,

    // 背景效果类型（Win11）
    BackdropType: windows.Mica,     // Mica 效果（推荐）
    // BackdropType: windows.Acrylic, // 亚克力模糊效果
    // BackdropType: windows.Tabbed,  // 选项卡式 Mica
},
```

效果对比：

| BackdropType | 效果 | 系统要求 |
|-------------|------|----------|
| `windows.Auto` | 自动选择最佳效果 | Win11 |
| `windows.Mica` | 微软推荐的半透明效果，与桌面壁纸融合 | Win11 22H2+ |
| `windows.Acrylic` | 高斯模糊半透明效果 | Win10 1803+ |
| `windows.Tabbed` | 类似浏览器标签页的 Mica 变体 | Win11 22H2+ |
| `windows.None` | 无特效（默认） | — |

#### 自定义主题色

#### WebView2 安装策略

WebView2 是 Windows 上的必需运行时。`webview2` 参数控制缺少时的行为：

```bash
# 构建时指定策略
wails build -webview2 download    # 自动下载安装（默认）
wails build -webview2 embed       # 将安装程序嵌入二进制（体积增大 ~150MB）
wails build -webview2 browser     # 提示用户用浏览器下载
wails build -webview2 error       # 报错退出
```

| 策略 | 说明 | 适用场景 |
|------|------|----------|
| `download` | 自动静默下载安装 | 默认推荐 |
| `embed` | 嵌入安装程序到 exe，离线可用 | 无网络环境 |
| `browser` | 弹窗引导用户浏览器下载 | 不想自动下载 |
| `error` | 直接报错退出 | 假定环境已安装 |

## 5. 自定义错误格式化

通过 `ErrorFormatter` 自定义绑定方法返回 error 时，前端收到的错误格式：

```go
// 默认行为：前端收到 error.Error() 字符串
// 自定义后：可以返回结构化错误

type AppError struct {
    Code    int    `json:"code"`
    Message string `json:"message"`
    Detail  string `json:"detail,omitempty"`
}

err := wails.Run(&options.App{
    // ...
    ErrorFormatter: func(err error) any {
        // 判断是否为自定义错误
        var appErr *AppError
        if errors.As(err, &appErr) {
            return appErr
        }
        // 通用错误
        return AppError{
            Code:    500,
            Message: err.Error(),
        }
    },
})
```

Go 端返回自定义错误：

```go
func (a *App) LoadData(id int) (Data, error) {
    data, err := a.db.Find(id)
    if err != nil {
        fmt.Printf("数据加载失败: %v\n", err)
        // 输出: 数据加载失败: record not found
        return Data{}, &AppError{
            Code:    404,
            Message: "数据不存在",
            Detail:  fmt.Sprintf("ID=%d 未找到记录", id),
        }
    }
    return data, nil
}
```

前端处理结构化错误：

```typescript
try {
    const data = await LoadData(999)
} catch (err: any) {
    console.log("错误码:", err.code)     // 输出: 错误码: 404
    console.log("错误信息:", err.message) // 输出: 错误信息: 数据不存在
    console.log("详细:", err.detail)     // 输出: 详细: ID=999 未找到记录
}
```

## 6. 安全性

### 6.1 AssetServer 中间件：请求拦截

```go
import (
    "net/http"
    "strings"
    "github.com/wailsapp/wails/v2/pkg/assetserver"
)

// 安全中间件 — 拦截敏感路径请求
func securityMiddleware(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        path := r.URL.Path

        // 阻止访问敏感路径
        blockedPaths := []string{"/api/admin", "/internal/", "/.env"}
        for _, blocked := range blockedPaths {
            if strings.HasPrefix(path, blocked) {
                fmt.Printf("拦截敏感请求: %s %s\n", r.Method, path)
                // 输出: 拦截敏感请求: GET /internal/config
                w.WriteHeader(http.StatusForbidden)
                w.Write([]byte("Access Denied"))
                return
            }
        }

        // 记录请求日志
        fmt.Printf("[%s] %s\n", r.Method, path)
        // 输出: [GET] /index.html
        // 输出: [GET] /assets/main.js

        next.ServeHTTP(w, r)
    })
}

// 使用
AssetServer: &assetserver.Options{
    Assets:     assets,
    Middleware: assetserver.ChainMiddleware(securityMiddleware),
},
```

### 6.2 Go 后端输入验证

```go
import (
    "fmt"
    "regexp"
    "strings"
    "path/filepath"
)

type Validator struct{}

// 验证用户名
func (v *Validator) ValidateUsername(username string) error {
    if len(username) < 3 || len(username) > 20 {
        return fmt.Errorf("用户名长度必须在 3-20 个字符之间")
    }
    matched, _ := regexp.MatchString(`^[a-zA-Z0-9_]+$`, username)
    if !matched {
        return fmt.Errorf("用户名只能包含字母、数字和下划线")
    }
    fmt.Printf("用户名验证通过: %s\n", username)
    // 输出: 用户名验证通过: wails_user
    return nil
}

// 验证文件路径 — 防止路径穿越攻击
func (v *Validator) SafeFilePath(baseDir, userPath string) (string, error) {
    // 清理路径
    cleanPath := filepath.Clean(userPath)
    fullPath := filepath.Join(baseDir, cleanPath)

    // 确保结果路径在基础目录内
    absBase, _ := filepath.Abs(baseDir)
    absTarget, _ := filepath.Abs(fullPath)

    if !strings.HasPrefix(absTarget, absBase) {
        fmt.Printf("路径穿越攻击被阻止: %s\n", userPath)
        // 输入 "../../etc/passwd" 时
        // 输出: 路径穿越攻击被阻止: ../../etc/passwd
        return "", fmt.Errorf("非法路径")
    }

    fmt.Printf("路径验证通过: %s\n", absTarget)
    // 输出: 路径验证通过: C:\Users\myapp\data\notes\test.txt
```

### 6.3 敏感数据安全存储

使用系统 Keyring 安全存储密码、Token 等敏感数据：

```bash
go get github.com/zalando/go-keyring
```

```go
import "github.com/zalando/go-keyring"

type SecureStorage struct{}

const serviceName = "com.mycompany.myapp"

// 安全保存凭证
func (s *SecureStorage) SetSecret(key, value string) error {
    err := keyring.Set(serviceName, key, value)
    if err != nil {
        return fmt.Errorf("保存凭证失败: %w", err)
    }
    fmt.Printf("凭证已安全存储: %s\n", key)
    // 输出: 凭证已安全存储: api_token
    return nil
}

// 读取凭证
func (s *SecureStorage) GetSecret(key string) (string, error) {
    secret, err := keyring.Get(serviceName, key)
    if err != nil {
        if err == keyring.ErrNotFound {
            fmt.Printf("凭证不存在: %s\n", key)
            // 输出: 凭证不存在: api_token
            return "", nil
        }
        return "", fmt.Errorf("读取凭证失败: %w", err)
    }
    fmt.Printf("凭证已读取: %s (长度=%d)\n", key, len(secret))
    // 输出: 凭证已读取: api_token (长度=64)
    return secret, nil
}

// 删除凭证
func (s *SecureStorage) DeleteSecret(key string) error {
    err := keyring.Delete(serviceName, key)
    if err != nil {
        return fmt.Errorf("删除凭证失败: %w", err)
    }
    fmt.Printf("凭证已删除: %s\n", key)
    // 输出: 凭证已删除: api_token
    return nil
}
```

## 9. Wails v3 新特性预览

> Wails v3 目前处于 Alpha 阶段，API 已趋于稳定，可用于生产环境。

### 9.1 核心 API 变化

```go
// ===== v2 — 声明式 =====
import "github.com/wailsapp/wails/v2"

err := wails.Run(&options.App{
    Title:  "My App",
    Width:  1024,
    Height: 768,
    Bind: []interface{}{
        &GreetService{},
    },
})

// ===== v3 — 过程式 =====
import "github.com/wailsapp/wails/v3/pkg/application"

app := application.New(application.Options{
    Name: "My App",
    Services: []application.Service{
        application.NewService(&GreetService{}),  // Bind → Services
    },
})

window := app.Window.NewWithOptions(application.WebviewWindowOptions{
    Title:  "My App",
    Width:  1024,
    Height: 768,
})

app.Run()
```

**v2 vs v3 关键差异**：

| 特性 | v2 | v3 |
|------|----|----|
| API 风格 | 声明式（一次性配置） | 过程式（逐步构建） |
| 绑定方式 | `Bind: []interface{}` | `Services: []application.Service` |
| 窗口 | 单窗口 | 多窗口 |
| 系统托盘 | 不支持 | 原生支持 |
| 构建系统 | Wails CLI 内置 | Taskfile（更灵活） |
| 绑定生成 | 反射 | 静态分析 |

### 9.2 多窗口支持

v3 最重要的新特性——原生多窗口：

```go
package main

import "github.com/wailsapp/wails/v3/pkg/application"

type App struct {
    app            *application.Application
    mainWindow     *application.WebviewWindow
    settingsWindow *application.WebviewWindow
}

func main() {
    myApp := &App{}

    myApp.app = application.New(application.Options{
        Name: "多窗口演示",
    })

    // 创建主窗口
    myApp.mainWindow = myApp.app.Window.NewWithOptions(application.WebviewWindowOptions{
        Name:   "main",       // 窗口名称（用于标识）
        Title:  "主窗口",
        Width:  1200,
        Height: 800,
    })

    // 创建设置窗口（初始隐藏）
    myApp.settingsWindow = myApp.app.Window.NewWithOptions(application.WebviewWindowOptions{
        Name:   "settings",
        Title:  "设置",
        Width:  600,
        Height: 400,
        Hidden: true,   // 初始隐藏
    })

    fmt.Println("主窗口和设置窗口已创建")
    // 输出: 主窗口和设置窗口已创建

    myApp.app.Run()
}

// 从主窗口打开设置窗口
func (a *App) ShowSettings() {
    if a.settingsWindow != nil {
        a.settingsWindow.Show()
        a.settingsWindow.SetFocus()
        fmt.Println("设置窗口已显示")
        // 输出: 设置窗口已显示
    }
}
```

### 9.4 系统托盘

v3 新增的系统托盘功能：

```go
package main

import (
    _ "embed"
    "log"
    "runtime"

    "github.com/wailsapp/wails/v3/pkg/application"
    "github.com/wailsapp/wails/v3/pkg/icons"
)

func main() {
    app := application.New(application.Options{
        Name: "托盘演示",
        Mac: application.MacOptions{
            ActivationPolicy: application.ActivationPolicyAccessory, // 不在 Dock 中显示
        },
    })

    // 创建窗口（初始隐藏）
    window := app.Window.NewWithOptions(application.WebviewWindowOptions{
        Width:       500,
        Height:      800,
        Frameless:   true,
        AlwaysOnTop: true,
        Hidden:      true,
        Windows: application.WindowsWindow{
            HiddenOnTaskbar: true, // Windows 任务栏中不显示
        },
    })

    // 创建系统托盘
    systray := app.SystemTray.New()

    // 设置图标（区分平台）
    if runtime.GOOS == "darwin" {
        systray.SetTemplateIcon(icons.SystrayMacTemplate) // macOS 模板图标
    } else {
        systray.SetDarkModeIcon(icons.SystrayDark) // 深色模式图标
        systray.SetIcon(icons.SystrayLight)         // 浅色模式图标
    }

    // 创建托盘菜单
    trayMenu := app.Menu.New()
    trayMenu.Add("显示窗口").OnClick(func(_ *application.Context) {
        window.Show()
        window.SetFocus()
        fmt.Println("通过托盘显示窗口")
        // 输出: 通过托盘显示窗口
    })
    trayMenu.AddSeparator()
    trayMenu.Add("退出").OnClick(func(_ *application.Context) {
        fmt.Println("通过托盘退出应用")
        // 输出: 通过托盘退出应用
        app.Quit()
    })
    systray.SetMenu(trayMenu)

    // 将窗口附加到托盘图标
    // 点击托盘图标时自动显示/隐藏窗口，失去焦点时自动隐藏
    systray.AttachWindow(window).WindowOffset(5)

    fmt.Println("系统托盘已创建")
    // 输出: 系统托盘已创建

    err := app.Run()
    if err != nil {
        log.Fatal(err)
    }
}
```

### 9.5 v2 → v3 迁移要点

| 变更项 | v2 | v3 |
|--------|----|----|
| 应用创建 | `wails.Run(&options.App{})` | `application.New(options)` + `app.Run()` |
| 绑定注册 | `Bind: []interface{}{app}` | `Services: []application.Service{application.NewService(app)}` |
| 窗口操作 | `runtime.WindowSetSize(ctx, w, h)` | `window.SetSize(w, h)` |
| 事件 | `runtime.EventsEmit(ctx, name, data)` | `app.Event.Emit(name, data)` |
| 菜单 | `menu.NewMenu()` | `app.Menu.New()` |
| 系统托盘 | 不支持 | `app.SystemTray.New()` |
| 构建 | `wails build` | `task build`（Taskfile） |

| 自定义协议 | `wails.json` 注册 scheme，macOS 用 `OnUrlOpen`，Windows 用启动参数 |
| 文件关联 | `wails.json` 配置扩展名，macOS 用 `OnFileOpen`，Windows 用 `os.Args` |
| 枚举绑定 | `EnumBind` 将 Go 枚举映射为 TS enum |
| 错误格式化 | `ErrorFormatter` 返回结构化错误对象 |
| Windows 特性 | Mica/Acrylic 效果、自定义主题色、WebView2 策略 |
| macOS 特性 | 标题栏样式、外观主题、关于面板、半透明效果 |
| 安全性 | AssetServer 中间件拦截、路径穿越防护、系统 Keyring 存储敏感数据 |
| 测试 | Go `testing` 包 + 表驱动测试、Vitest + Mock Wails 运行时 |
| 构建 | `wails build`、ldflags 注入版本、NSIS 安装包、UPX 压缩 |
| CI/CD | GitHub Actions + `wails-build-action` 多平台自动构建发布 |
| Wails v3 | 多窗口、系统托盘、Services、过程式 API、Taskfile 构建 |
