# Go 语言介绍与环境搭建

## 为什么选择 Go

Go 是 Google 在 2009 年推出的一门 **静态强类型、编译型** 语言，目标是：

- **简单**：语法简洁，学习成本低
- **高性能**：编译为机器码，执行速度快
- **并发友好**：原生支持 goroutine + channel
- **工程化强**：自带格式化、测试、依赖管理

**Go 的典型应用场景**

- 后端 Web 服务（API、微服务）
- 高并发系统（网关、消息系统）
- 云原生 / Kubernetes / Docker
- 中间件（etcd、Prometheus）

## Go 语言特点速览

| 特性       | 说明                        |
| ---------- | --------------------------- |
| 编译型     | 直接生成可执行文件          |
| 跨平台     | 一次编写，多平台编译        |
| 垃圾回收   | 内置 GC，解放内存管理       |
| 并发模型   | goroutine + channel         |
| 工具链完善 | go build / go test / go fmt |

## Go 环境安装

官网下载：

1. 打开官网 https://go.dev/dl/

2. 下载对应系统版本

   - Windows：`go1.xx.x.windows-amd64.msi`
   - macOS：`go1.xx.x.darwin-amd64.pkg`
   - Linux：`go1.xx.x.linux-amd64.tar.gz`

3. 安装完成后，打开终端验证：

   ```bash
   go version
   
   # 输出go版本
   # go version go1.24.6 windows/amd64
   ```

## Go 环境变量说明

安装完成后，一般会自动配置：

- **GOROOT**：Go 安装目录（通常不用手动设置）
- **GOPATH**：Go 工作区（1.11+ 已非必须）
- **PATH**：需要包含 Go 的 bin 目录

查看环境：

```bash
go env
```

> **建议**：使用 Go Modules，不再依赖 GOPATH 结构
>
> 这样 Go 项目可以放在任意目录，不需要再放到 `$GOPATH/src` 下面；
> 依赖由 `go.mod / go.sum` 管理，而不是靠目录结构。

Go Modules 的常用命令

```bash
go mod init 项目名      # 初始化模块
go mod tidy            # 自动整理依赖
go mod download        # 下载依赖
go get xxx 			   # 下载某个依赖
go list -m all         # 查看所有依赖
```

## Hello World

```bash
mkdir go-learn
cd go-learn
# 初始化模块
go mod init go-learn
```

新建 `main.go`

```go
package main   // 包名，程序入口必须是 main

import "fmt"   // 引入标准库

func main() {  // 程序入口函数
	fmt.Println("Hello, Go!")
}
```

运行

```go
go run main.go
```
