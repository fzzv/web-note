# 链表

## 链表界最实用的作弊技巧——**哨兵节点 (Dummy Node)**

比如第21题：合并两个有序链表

将两个升序链表合并为一个新的 **升序** 链表并返回。新链表是通过拼接给定的两个链表的所有节点组成的。

**输入：** `l1 = [1, 2, 4]`, `l2 = [1, 3, 4]`

**输出：** `[1, 1, 2, 3, 4, 4]`

**遇到的困难：** 我们要生成一个新链表。但是新链表的第一个节点（头节点）应该是谁呢？ 是 `l1` 的第一个？还是 `l2` 的第一个？ 我们需要写一堆 `if/else` 来判断谁小谁大，还要专门记录头节点，很麻烦。

**解决方案：** 我们要么先买个“假头”放在那。

```go
dummy := &ListNode{Val: -1} // 这是一个假的占位符
current := dummy            // 我们的操作指针从这个假头开始
```

不管 `l1` 和 `l2` 谁先来，我们都统统接在 `current` 后面。 最后，我们只要返回 `dummy.Next` 就像是把这一长串提起来，把“假头”扔掉，就是完美的结果了。

```go
func mergeTwoLists(list1 *ListNode, list2 *ListNode) *ListNode {
    // 1. 创建哨兵节点 (dummy)
    dummy := &ListNode{Val: -1}
    
    // 2. 创建一个移动指针 (cur)，一开始指向 dummy
    cur := dummy

    // 3. 当 list1 和 list2 都不为空时，比较它们的值
    for list1 != nil && list2 != nil {
        // 谁小，cur.Next 就指向谁，然后那个指针后移
        if list1.Val < list2.Val {
            cur.Next = list1
            list1 = list1.Next
        } else {
            cur.Next = list2
            list2 = list2.Next
        }
        // 别忘了，不管连了谁，我们的 cur 指针都要往前走一步，准备接下一个
        cur = cur.Next
    }

    // 4. 处理剩下的链表
    // 检查 list1 是否还有剩？如果有，接上 list1。
    // 否则，接上 list2 (不管 list2 是不是空的，接上也没错)
    if list1 != nil {
        cur.Next = list1
    } else {
        cur.Next = list2
    }

    // 5. 返回真正的头节点（跳过 dummy）
    return dummy.Next
}
```
