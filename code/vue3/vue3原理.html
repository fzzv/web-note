Vue 3 核心架构全景

Vue 3 是一个模块化的前端框架。从宏观角度看，它由三个核心部分协同工作： 编译模块 (Compiler) 将模板转化为渲染函数， 响应式模块 (Reactivity) 追踪数据变化， 运行时模块 (Runtime) 负责创建和更新虚拟 DOM。

Vue 3 性能优化目标

  • ✓ 更小的体积 (Tree-shaking): 模块化设计使得未使用的 API 不会被打包。
  • ✓ 更快的挂载 (Proxy): 消除 Vue 2 `Object.defineProperty` 的初始化性能瓶颈。
  • ✓ 编译优化 (Hoist & PatchFlags): 编译器分析模板，告诉运行时哪些部分是静态的，哪些是动态的。

编译系统 (The Compiler)

Vue 3 编译器的主要任务是将开发者编写的 HTML 模板 (Template) 转换为 JavaScript 渲染函数 (Render Function)。 这个过程包含三个主要步骤：Parse (解析), Transform (转换), 和 Codegen (生成)。

💡 核心优化：PatchFlags

在 AST 转换阶段，Vue 3 会分析节点是否依赖动态数据。如果是动态的，会打上标记 (PatchFlag)。 在 Diff 阶段，运行时只需对比带有 PatchFlag 的节点，跳过静态节点。这就是 Vue 3 比 Vue 2 快的主要原因之一。

运行时 (Runtime & Renderer)

Runtime 的核心是 Renderer (渲染器)。它利用 `render` 函数返回的 VNode (虚拟节点) 树，通过 DOM API 将其渲染到浏览器页面上。

VNode 结构示意

const vnode = { type: 'div', props: { id: 'app' }, children: [ { type: 'p', children: 'Hello' } ], shapeFlag: 17, // Bitmask info patchFlag: 0, // Optimization info el: null // Ref to real DOM }
虚拟 DOM 仅仅是一个普通的 JavaScript 对象。相比操作真实的 heavy DOM 对象，操作 VNode 极其廉价。

自定义渲染器 (Custom Renderer)

Vue 3 的 `runtime-core` 不依赖于浏览器。通过提供不同的节点操作 API (nodeOps)，Vue 可以渲染到 Canvas、Native (Weex/UniApp) 或终端。

createRenderer({ createElement: (tag) => ... patchProp: (el, key, val) => ... insert: (el, parent) => ... })

Diff 算法可视化 (The Patch Process)

当数据变化导致 VNode 树更新时，Vue 需要对比新旧子节点列表 (`patchKeyedChildren`)。 Vue 3 采用了 快速 Diff 算法 (Fast Diff)，结合了双端对比和最长递增子序列 (LIS) 算法来最小化 DOM 移动操作。
