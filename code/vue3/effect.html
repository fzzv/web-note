全局指针与副作用栈 (Global Cursor & Effect Stack)

在响应式系统中，当一个副作用函数 (Effect) 执行时，系统必须知道“当前是谁在运行”，以便将它与由于读取数据而触发的依赖关联起来。 当组件嵌套时（Effect 嵌套 Effect），我们需要一个 Stack (栈) 来保存父级 Effect，确保子级执行完后能正确恢复父级环境。

为什么需要栈? 如果没有栈，当子组件 Effect 执行完毕将 `activeEffect` 置为 `null` 时，父组件后续的依赖收集将失败。栈确保了指针能回溯到父级环境。

调度器与去重 (Schedulers & Deduplication)

Vue 的响应式系统不会在数据变化时立即执行副作用，而是将它们推入一个 Queue (微任务队列)。 这允许系统在同一 "Tick" 内对同一数据的多次修改进行去重，只执行最后一次更新，极大地优化了性能。

清除副作用 (Effect Cleanup)

当副作用包含异步操作（如 API 请求）或监听器时，如果不及时清理，会导致 竞态问题 (Race Conditions) 或内存泄漏。 Vue 提供了 `onInvalidate` 回调，在副作用重新执行前或组件销毁时触发。
